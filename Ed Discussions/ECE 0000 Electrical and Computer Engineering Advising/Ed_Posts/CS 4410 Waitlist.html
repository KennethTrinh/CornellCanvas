As a general matter, being on the waitlist does not automatically enroll you in a class when space opens up. Assuming the class is configured to lock when it becomes full for the first time, either the registrar would need to add you or the dept. offering the class would need to send you a PIN. If the class is not set to lock, then when a space opens up it is a free-for-all, and the first person to enroll gets the slot, regardless of where they are on the waitlist, or whether they are on the waitlist at all. Yes, that is unexpected behavior that arguably defeats the purpose of a waitlist. Usually when the waitlist is enabled for a class, the class is set to lock upon initial fill-up, but no guarantees. It sounds like CS 4410 locked. Needless to say, I personally feel that the waitlist system that Cornell uses has room for improvement, and I believe it will be eventually. But that is how it works now.
