This is almost always an issue with your SSH Agent. (Not your SSH Keys. If the command ssh -T git@github.coecis.cornell.edu successfully authenticates after you run it in Git Bash (Windows) or Terminal (Mac), then your keys are correct. Please do not redo your keys.)

Please see the Configure SSH Agent instructions in Lab 1 to fix issues with the SSH Agent.
