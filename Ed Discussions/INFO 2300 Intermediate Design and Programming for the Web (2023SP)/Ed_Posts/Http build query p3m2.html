I'm sorry that you were told to use a button. While I do not know the details of what has happened here, I do know that the project requirement discourage a button for things that would not have a button in other catalogs (i.e. design patterns). You should not have been told something that contradicts the assignment's requirements. From the Project 3 requirements:
A common mistake on this project is to have “submit” buttons everywhere on your site.
Not many catalogs have a button to go to the details page. They often have buttons for "Add to Cart," but not "Go to Details." A hyperlink is used with query string parameters instead. You should probably use query string parameters. I know this is frustrating. However, the good news is that updating the HTML code for the view all page, is only a small portion of the milestone 2 requirements and doesn't require redoing the entire milestone. I am happy to help you in my office hours to make this update. Please feel free to come and see me if this answer isn't satisfactory. I'm more than happy to help you and make changes where necessary.
