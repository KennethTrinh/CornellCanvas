To use the terminal, you do similar steps:
git add . to stage all the files
git commit -m "your msg" to make a commit, and where the your msg is how you want to name the commit
git push to push to the remote repo
However, you might just get the same issue with access rights. Try to redo the ssh key steps in the initial lab setup. Also, keep in mind the stuff we do in class can't be pushed - only your labs and projects.

Good answer from Ming, but just to add: If you find that this issue occurs only on VSCode but not on the command line (Terminal), then you may want to try following the steps from the beginning of the semester to ensure that you have the correct ssh key configured in your VSCode settings. Usually I've found that people with this issue have more than one ssh key on their systems, with one that they previously used for regular ssh access (e.g. to github.com), and another one they created for this class (for github.coecis.cornell.edu). Because of this, you may have only registered the first key on VSCode, when you actually needed to register the second one.
