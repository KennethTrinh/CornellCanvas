In the announcement, it says not to hard code or concatenate in SQL queries.

I saved the tag id as a variable (using $_GET) rather than in an array as shown in the example and then set the where clause to equal 'WHERE tags.id = ' . tag_id_variable. Would this be acceptable, or would I need to do it the same way as stated in the example? No, this is not acceptable because it is unsecure. Please review the class notes for "SQL Security in PHP"

Never trust data from $_GET . If you take something from $_GET and concatenate it into an SQL query, you are opening your website up to SQL injection. This is really bad. You must use parameters markers for your $tag_id_variable .

The only place where concatenation is acceptable for SQL is concatenating SQL clauses. Most everything else is likely to be insecure.
