You need not built all the functionality, so long as you met the project's requirements. The answer depends on how you view the project's requirements. They are designed to be flexible to permit students to implement projects that are personally motivating. The requirement for tags is simply a one-to-many or many-to-many table relationship requirement. That means you can implement convention tags, as we explored in class with the seed activity. However, if each user has a cart, then adding items to a cart is essentially tagging items for a user. This is a bit confusing, and so if you'd rather built a shopping website with conventional tags and leave out the "add to cart" button, that also works! e
