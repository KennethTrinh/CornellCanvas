htmlspecialchars() is required when echoing input information from the user onto the screen, as it filters out the meaning some special characters in html (such as <style> </style>), and outputs the exact raw text that was inputted. One thing you can try to test out the effect of this try to input something into a form textbox with <style> tags around it, such as changing the color. As was demonstrated in the HTML forms lecture, without htmlspecialchars(), you will notice that the style of the input-text echoed will have the style you added into the textbox instead of what you have coded in your codebase. With htmlspecialchars(), the echoed text will appear as you have intended in your code. The style tags is just one example- there are much more malicious things a user can input to test whether or not you have htmlspecialchars() or not.
