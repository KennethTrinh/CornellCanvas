An algorithmically implemented query is when you query all records and then use conditional statements inside the foreach loop to either display or not display a record. It's really inefficient.

If you want each entry to have its tags visible on the view all page, you will need to have another foreach loop nested and query the database again. This is how modern web frameworks handle it. This is because many websites may have millions of tags. But each entry only has a few tags. If you query the entire list of tags and then try and store them whether than querying again, it would consume a lot of memory. You probably would have been better off making the tags query... like you're doing.
