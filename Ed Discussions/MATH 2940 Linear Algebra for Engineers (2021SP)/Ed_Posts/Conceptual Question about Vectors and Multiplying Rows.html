In a linear system, each column can be treated as a vector. For example, take 1.4.22: v_1=\left[\begin{matrix}\ \ \ 0\\\ \ \ 0\\-2\end{matrix}\right],\ v_2=\left[\begin{matrix}\ \ \ 0\\-3\\\ \ \ 8\end{matrix}\right],\ v_3=\left[\begin{matrix}\ \ \ 4\\-1\\-5\end{matrix}\right] If we were to write these vectors into one linear system coefficient matrix, it would become: \left[\begin{matrix}\ \ \ 0&\ \ \ 0&\ \ \ 4\\\ \ \ 0&-3&-1\\-2&\ \ \ 8&-5\end{matrix}\right] My question is how are we able to scale a row if scaling a row alters the proportions of a vector? For example, if we multiply row 2 of this coefficient matrix by 10, we get: \left[\begin{matrix}\ \ \ 0&\ \ \ 0&\ \ \ 4\\\ \ \ 0&-30&-10\\-2&\ \ \ 8&-5\end{matrix}\right] And clearly \left[\begin{matrix}\ \ \ 0\\-30\\\ \ \ 8\end{matrix}\right] is not a scalar multiple of the original \left[\begin{matrix}\ \ \ 0\\-3\\\ \ \ 8\end{matrix}\right]

Scaling the second row by 10 will change the set of right-hand-sides that have solutions (i.e. $Span\{v_1,v_2,v_3\}$ will be different from $Span\{w_1,w_2,w_3\}$ where $w_i$ comes from multiplying the second entry of $v_i$ by 10) but \left[\begin{matrix}v_1&v_2&v_3&|&b\end{matrix}\right] will have the same solution set as \left[\begin{matrix}w_1&w_2&w_3&|&c\end{matrix}\right] where b\ =\ \left[\begin{matrix}b_1\\b_2\\b_3\end{matrix}\right]\ and\ \ c\ =\ \left[\begin{matrix}b_1\\10b_2\\b_3\end{matrix}\right] When you are row reducing an augmented matrix, you are also adjusting the right hand side values so that the solution set stays the same. You will see more about this in Chapter 4 when we see how row operations change column spaces but preserve row spaces.
