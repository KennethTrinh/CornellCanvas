When a new-expression is "evaluated", the value returned is the object itself, essentially a reference/pointer, which is stored in the variable.

If the new expression is:

Object o= new Object();

then the variable s (think of it as a box with a value inside of it) will contain the reference/pointer to the object o, for example o@5 (which is a representation of object o at memory location 5). The important thing is that the variable contains a reference to the object, not the object itself.
