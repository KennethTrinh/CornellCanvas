I would need to see the class header, but in general, when we have "addActionListener(this)" the frame itself implements interface ActionListener. So that's why we're able to add "this" (an object of a class implementing ActionListener) as an ActionListener for the button. Then, somewhere else, that class would need to implement actionPerformed (the only abstract method in the interface). The anonymous function is a shorthand for implementing the ActionListener interface. Since the interface has only one abstract method, that's when we can have an anonymous function with code to be executed. So what's inside parentheses automatically becomes the implementation of the actionPerformed method. Here's what the anonymous function does: 1. use the code inside the parentheses to implement method actionPerformed, which is the only abstract method of interface ActionListener. 2. By implementing the interface, we have thus created class C. We return an object of class C as the value of the expression, which is then added as the ActionListener. Sorry if this was a bit long. This process is summarized in the answer key for SP18 (I think) more concisely than how I put it.
