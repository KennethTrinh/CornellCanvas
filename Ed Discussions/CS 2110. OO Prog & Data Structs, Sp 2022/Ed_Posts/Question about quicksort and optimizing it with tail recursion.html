Yes, you are correct.

But this version of yours, with the smaller partition being sorted first and the larger partition being sorted last, using tail recursion, is another way to get logarithmic depth in general.

Unfortunately, Java does not implement tail recursion. One here's about them working on it, but that has been for some time. I think that one of the issues is exception handling. As you know, when an exception is thrown, it is sort of thrown up the call stack, and one frame on the call stack may catch it. That limits tail recursion somewhat.

But YOU can implement the two tail recursive call using recursion yourself.! This page in JavaHyperText shows you how:

https://www.cs.cornell.edu/courses/JavaAndDS/files/recursionTail.pdf
