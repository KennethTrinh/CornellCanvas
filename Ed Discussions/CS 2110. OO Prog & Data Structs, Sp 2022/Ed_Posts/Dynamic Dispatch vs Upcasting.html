From my understanding, dynamic dispatch happens when the compiler choses which method to run at runtime if the methods are overloaded, from parent to subclasses. Upcasting also happens automatically, when the compiler automatically casts an object up its hierarchy. Do they both use the bottom up rule? Are they kind of the same thing or two very different concepts?

At runtime, in order to see what method a call m(...) refers to, ALWAYS use the bottom up rule.

If the call starts with super.m(...), then use the bottom up rule starting at the partition above the one in which the call appears.

The above has NOTHING to do with casting or coercion, which is automatic upcasting by Java. They just determine the (static) type of an expression a compiletime.

SEPARATE COMPILE TIME FROM RUNTIME IN YOUR MIND

The static type of the expression new CAT(...) is Cat

The static type of the expression (Animal)(new Cat()) is Animal.
