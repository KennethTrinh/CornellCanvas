For the binary search example in lecture 11, I'm a bit confused about why the worst case time is log(b.length), especially when considering the case where there's an array [0,1,2,3,4,5,6,7], and v = 7.

Wouldn't this case have log(b.length)+1 iterations?

Or does this case not satisfy the condition that b[0..i] <= v < b[i+1..], since the values of an empty array cannot be greater than v?

Time complexity O(log(b.length)) is the same as O(log(b.length)+1).

It may be easier to think of the function that grows the fastest. If the function does one extra iteration, the time will barely change. Doing log(b.length) iterations will require much more time as b.length grows.
