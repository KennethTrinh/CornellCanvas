1) An abstract method is a method defined that you know Must be overrided.

2) A class is abstract when you don't want a user to create an object of that class, but to be just more of a template, so let's say you have Abstract Class Animal, we have subclasses Cat,Dog, Cow. We want Cat and Dog and Cow to have all the methods that Animal does but that doesn't mean we want to be able to create an animal, we want to be more specific.

3) You can't create instances of an abstract class because you're using Java period. Creating an instance of Shape or Animal in my case is too unspecific, and useless. Circle is more of a SPECIFIC shape therefore we want to be able to use circles, we never say "I'm going to make the Animal make a sound", Which sound? Which animal? We know all animals make a sound though, so we have an abstract method called makeSound() in the Animal class for example.

I'm going to find the area of the shape.

Which Shape? How do you find the area? Which equation do you use? But we know all shapes have a way to find their area so we create a findArea() abstract method in shape.
