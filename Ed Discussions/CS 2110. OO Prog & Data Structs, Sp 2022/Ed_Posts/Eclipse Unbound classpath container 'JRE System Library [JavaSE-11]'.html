Title: Eclipse: Unbound classpath container: 'JRE System Library [JavaSE-11]'

Author: David Gries (admin)

Date: 2021-12-29T02:15:40.51997+11:00

Category: General

Vote Count: 0

Errors:

The project cannot be built until build path errors are resolved

Unbound classpath container: 'JRE System Library [JavaSE-11]'

Problem: Eclipse is not using the most up to date JRE.

Solution 1: From Package Explorer, right click any project and choose Build Path > Configure Build Path. Click the Modulepath heading and choose "Add Library" Choose "JRE System Library" then click "Next" From the next window, check to make sure that the Workspace default JRE is pointed to jdk 11. If this isn't the case, click "Installed JREs" and click "Search" Then navigate to the newly installed JRE in your file explorer and click "Select Folder" Click "Apply and Close"

At this point, the 2 errors should disappear from all previous projects.

Solution 2:

You can also change the default JRE Eclipse uses when creating a project:

Go to Preferences > Java > Installed JREs

and then make sure Java SE 11 is checked.

Hit 'Apply and Close' and that should do it!
