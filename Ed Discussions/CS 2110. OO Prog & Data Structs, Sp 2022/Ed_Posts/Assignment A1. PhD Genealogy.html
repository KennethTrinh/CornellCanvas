1. When I create a JUnit testing class and then put in an assertEquals or assertThrows statement, I get an error.

Press your mouse over the red x before a line with an assertEquals or assertThrows call that shows an error. A window will pop up suggesting the need for an import statement. Select that and let Eclipse put in the import statement.

2. Should I put files in the default package? No.

We want both PhD.java and PhDTest.java to be in package a1.

3. How many times can I submit on the CMS ? Submit as often as you like. Only the last one is graded.

BUT DO NOT take this as an opportunity to submit every 3 minutes, submitting 15-20 times. Submit when you know it is done, but if later you see how to do something better, submit again.

4. I get a warning about my method specification because I am not using @param or @return . What do I do?

Ignore the warnings and delete those @param things for now. We don't need them.

Read about method specifications in our Style Guidelines: Visit JavaHyperText page and click "Style Guide" in the left part of the top navigation bar. 5. Should we check for situations like a PhD is their own advisor or someone got their PhD in a year that is < 0 ?

No. Please implement the spec of each method exactly; do nothing more, nothing less.

We are not attempting to create a completely realistic class to be used in a real life situation.

We are attempting to teach you about OO in Java with a minimum amount of your time.

6. Can we add other methods to class PhD ? Do not add other methods. There is no need for them.

7. If I want to call constructor 1 from constructor 2, I can't put assert statements at the beginning of constructor 2. What do I do? If you are not contemplating calling one constructor from another, which hasn't been taught as of this writing, no need to read further.

But the answer to this question appears in note #11.

8. Suppose my field for the number of advisees is nadv. Since nadv is private, from PhD object p (say), how can I reference/change nadv, the number of advisees in PhD object q?

Just do it; you will see that it works. Just use q.nadv to reference q's field nadv.

Any method in class PhD can reference private fields and methods of any object of class PhD. 9. May I use fields in PhDTest to reduce the number of declarations needed in methods in this class?

No, don’t do that, for at least three reasons.

  • You don’t know enough about JUnit testing to do this right. For example, you don’t know the order in which methods are called.

  • Such fields often create interactions between testing procedures, but those testing procedures should be completely independent of each other.

  • A declaration may be far from its use, making it more difficult to understand the methods.

    • Remember, once you write a declaration-assignment, you can easily copy-and-paste it into another method.

    10. How can I write this in Java? "this PhD and PhD p are different"?

    Keyword this is a pointer to the object in which it appears. You will learn this soon. So you can write this expression as

    this != p
