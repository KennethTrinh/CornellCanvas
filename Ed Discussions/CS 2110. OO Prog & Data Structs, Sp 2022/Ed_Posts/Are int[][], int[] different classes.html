1. Yes, the folders all represent objects of either int[][] or int[]. 2. You should consider an "array" to be an object, whose behavior is just like any other object; it is located in some place in memory, where the variable allows us to access it in its place in memory. An array is not solely the variable e itself, nor is simply the elements contained within the array, because we can also do things like e.length which is a field of the object. 3. int[][] and int[] are indeed class names, but they're defined by Java in a special way, so that we can add more brackets if we want more dimensions, thereby making them extensible.
