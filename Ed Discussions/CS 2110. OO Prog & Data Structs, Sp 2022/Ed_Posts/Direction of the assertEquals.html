Does it matter if testing is written as

1) assertEquals(test.isSibling(test1), true);

OR

2) assertEquals(true, test.isSibling(test1));

The definition is assertEquals(expected val, computed val);

so use it only that way. If it succeeds, it doesn't matter, but if it fails, the error message won't make sense if wrote in wrongly.
