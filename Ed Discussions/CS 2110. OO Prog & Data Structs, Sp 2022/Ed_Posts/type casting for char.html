If char is more narrow than int, why does Java still implicitly cast an int to char? for example, char c = 97 gives c the value 'a'

char is narrower than int, so casting to a char from an int could potentially result in data loss (for example, try char c = 100000). We convert here because the type is char, and every char corresponds to an integer value (its ASCII value to be exact).
