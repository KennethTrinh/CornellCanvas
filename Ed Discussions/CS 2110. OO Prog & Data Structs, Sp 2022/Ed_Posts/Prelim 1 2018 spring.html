In this question, why is it necessary for Dress 1 to be abstract? Why can't a non-abstract class have an abstract method when an abstract class can have a non-abstract method?

Author: Andrew Chen (staff)

Vote Count: 0

The whole idea of an abstract class is to define a class that will not be instantiated. So you will not create a class Dress1, but rather you will create a subclass that inherits Dress1's methods/fields and defines the abstract methods. We think of abstract classes (and interfaces for that matter) as contracts of more general types. They are two general so we don't want to create an object of that type, but we create a template for child classes to base their definition off of.

Why abstract classes can declare and implement non-abstract methods: Maybe you have some methods which you want to allow the subclasses to define. But for others maybe you have an implementation that you want to define for the child classes.
