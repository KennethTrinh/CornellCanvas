That in general doesn't always fail. The slide is referring specifically to the case with the //? after it (i.e. IAccnt w= (IAccnt) (new Accnt()); ). This fails at runtime because even though it is possible for a value with static type Accnt to have an IAccnt partition, since we used the Accnt constructor, the object has dynamic type Accnt, meaning it does not have an IAccnt partition, so casting would throw a ClassCastException.
