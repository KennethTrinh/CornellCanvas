I was wondering why the following statement from SP16A is false:

Method m() processes a list of size n using nested for loops. Therefore, the run time of m() is O(n^2).

It all depends on what the loops do. Consider the loops shown below. The inner loop takes constant time since it always executes 5 iterations.

for (int k= 0; k < n; k= k+1) { for (int j= 0; j < 5; j= j+1) { t= t+1; } }
