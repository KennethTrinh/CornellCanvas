Would super.method() only be useful when there are 2 methods with the same name in both the super and sub classes?

If you are simply trying to access a uniquely named function in the superclass, can't you simply call a function without super and by the inside out rule, the function in the superclass would automatically be called?

Author: Andrew Chen (staff)

super.method() would be absolutely necessary when you are overriding a method from the parent class and want to call the parent class's version of that method. Not putting the super. would make the method recursive.

Other methods that are not overridden are inherited , that is, they exist in the subclass automatically, so you would not use super.
