The internals of an ArrayList in Java are essentially an array that the user can add items to, placed into the next open index of the array. If the capacity of the array is reached, a new one with higher capacity is created and the elements from the previous array are copied into it. This means that creating an empty ArrayList only involves initializing a new array with the length being the specified capacity (by default, the initial capacity is 10 unless specified). This is because initialization of an array in Java, by the JVM specification, requires all elements in the array to be initialized to their default values, which takes O(n) time. This means that overall, the time complexity of creating an empty ArrayList is O(n) with respect to the initial capacity. You can see the constructors of the ArrayList class in Java 8 below. Notice how all that is done is create a new array of a specified capacity. The super constructor is empty and does not affect this method.
