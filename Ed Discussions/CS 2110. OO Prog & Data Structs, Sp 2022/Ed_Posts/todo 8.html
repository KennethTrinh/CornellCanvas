1. Is a duplicate of #1479.

2. You have to implement the functionality that draws the game in either the active or inactive state. That handout is simply describing the fact that --if you've done it right-- the inactive state will return in 15 seconds by default. There's no code that you need to write related to seconds or 15 or whatever here. Just follow the specification of TODO 5 ( timeout ) and it will work correctly.

3. It sounds like you need to debug your TODO 7 and 8. For example, you could add some print statements to TODO 7 to indicate what the coordinates are, what the circle's center and radius are, and then check by hand whether it should have been a hit or not. As the student answer points out, the root cause could be drawing the circle incorrectly in TODO 6.
