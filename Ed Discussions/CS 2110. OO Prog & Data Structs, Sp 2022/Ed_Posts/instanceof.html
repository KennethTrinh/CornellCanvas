From my understanding, it depends on if you want to compare/check the object's status in general terms or specific terms.

For example, if obj A is a Cornell University freshman:

Using instanceof will check if the object belongs to a more broader group (ex: if A is a student). Using getClass() will check what its most specific identity is (ex: A is Cornell University freshman).

If you want to determine whether the object can be cast to some class or interface, the instanceof will do.

If you want to know the class with which the object was created, used getClass().

Generally when writing a method equals, look at the specification! It should give enough information for you to determine which to use.
