(Cat)( new Animal() ) will throw a ClassCastException during runtime?

Author: Andrew Chen (staff)

Vote Count: 1

You cannot downcast to a Cat if the instance is of class Animal. An Animal is not the same as a Cat .

So this is legal:

Animal a= new Cat();

Cat c2= (Cat) a;

But this is not:

Animal a= new Animal();

Cat c2= (Cat) a;

Notice how the reference type/static type in both scenarios is Animal. However, whether or not downcasting is legal depends on whether or not the dynamic type is actually an Animal or a Cat. Basically, is the variable a pointing to an Animal or a Cat? Search up "dynamic type" in JHT.
