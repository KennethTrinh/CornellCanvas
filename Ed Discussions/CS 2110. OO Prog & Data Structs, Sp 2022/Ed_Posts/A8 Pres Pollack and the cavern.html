    4. What you can and cannot do :

    4A. Do NOT use static fields (unless they are constants, which never change). Our grading program could run several caverns simultaneously, using different threads, and having a static field that can change will be disastrous

    4B. Declare as many fields as you want. Write as many methods, static or non-static, as you wish. We suggest commenting fields appropriately and specifying method carefully. The staff is instructed not to help with fields or methods that are not appropriately defined (specified).

    4C. We told you not to make the two main methods, find() and flee(), recursive for the following reason. For maximum flexibility, write a dfs walk (or code for flee) in a separate method. Then, call that method from find() (or flee()). Why? First, you may need some initialization of fields or other things to record the actual dfs walk that takes place. Second, you may be writing two dfs walks. The first is the basic one, the second is an optimized one. If you don't get the optimized one working properly, just before submitting, fix find() to call the first one, which works. You should always be programming to provide yourself with maximum flexibility, especially in a program like this, where you may be trying several different strategies.

    6. Worried about how much time Pres Pollack takes to flee a certain cavern ?

    Watching Pres Pollack in GUI mode is not a good way to figure out the time she is taking to flee. It all depends on the speed slider, and you can't speed it up enough.

    Instead, do the following to calculate and print the elapsed time of flee in the headless mode.

    A. Put this field and method into class Pollack (you can leave them there):

    private long startTime= 0; // start time in milliseconds

    /** Return the time from start time till when this method is called, in milliseconds */ public long elapsedTime() { return System.currentTimeMillis() - startTime; }

    B. Insert the following statement as the first statement of method flee(...):

    startTime= System.currentTimeMillis();

    C. Insert the following statement as the last statement of method flee(...):

    System.out.println("Elapsed time: " + elapsedTime() / 1000.00 + " seconds");

    D.Then run in headless mode with whichever seed you are worried about.

    Be sure to comment it out before you submit.
