I believe that D could work. If you put just A into the Stack and then pop it out and push it to Q, A is added to the queue. Remember that an element can be popped from S at any time. Following this method, if you add just B into the empty Stack (since A was popped out) and then pop B out and push B to Q, then the queue is (A,B). A stack pops off the top value (the most recently added) so D would be the case if all values are popped off as soon as they are added to the stack. If the values are only popped off after all have been added, S[A,B,C,D,E], then Q would have them in the opposite order, which is choice A, since E gets popped off, then D, then C.... Choice B can occur if D and E are popped off immediately after being added, then A, B and C are left on the stack, S[C,B,A], then popped off top to bottom (C then B then A). C is not possible because when E is added to the queue, B is still in the stack, so there is no way A will be popped off first if B is still above it. Hope this helps!
