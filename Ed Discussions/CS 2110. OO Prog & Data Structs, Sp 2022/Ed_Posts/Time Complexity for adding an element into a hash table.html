When we are using a hash table for a set of objects, I do not understand how adding and removing an element to the hash table has expected time O(1).

For adding, you would first have to search through the linked list in the bucket to determine whether the element is in there, which is O(n), and then add the element if it is not there.

For removing, you would also have to search through the linked list in the bucket again, and then remove the element if it is there.

I understand that it would be O(1) if there were no collisions, but then why would we be creating a linked list to hold elements if there were no collisions?
