Yes, and I will add onto that with this example: int x; This is a declaration of variable x, indicating that it can hold a value of type int. -------- x= 25 + 1; This is an assignment statement . Its execution evaluates the expression 25+1 to yield the value 26 and stores that value in x. ------- int x= 25+1; One can combine a declaration and an initial assignment to it, as shown here. The "int x" part declares variable x of type int. The "x= 25+1" part assigns it the initial value 26. From JHT: "[A variable declaration] has the form < type > < variable-name >."

Declaring a variable allocates a new piece of memory for storing that variable. Assigning value to a variable just replaces the value on that variable and doesn't require new memory spaces. In Java, declaring a variable often gives it a default value though.

Declaring a variable is the same no matter the type. Arrays themselves are also types. Just like you have int, float, double, you also have int[], float[], double[]. ------- So the declaration of an array of ints would be just: int[] x; --------- We can create an array, with its elements set to their default values, and then assign it to variable x: x= new int[3]; --------- Combined declaration, creating and then assignment of new array to variable x: int[] x= new int[3]; --------- I think you'll benefit from reading this as well! It goes more into depth about creating/initializing the elements of an array, and also array initializers. https://www.cs.cornell.edu/courses/JavaAndDS/files/arrays.pdf
