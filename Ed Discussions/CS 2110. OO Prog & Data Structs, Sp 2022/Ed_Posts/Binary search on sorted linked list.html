Yes, adding on: what you described with the time complexities only applies to arrays and ArrayLists, or anything in which querying the collection by index takes O(1) time. So in an array, we can simply write arr[i] for any i and we get the element at that index in O(1) time.

In a LinkedList, as you saw in A4, you have to iterate (from the head or the tail) to get to that element at index i.

To get to node I. in a linked list takes time O(i) -- takes time linear in i.

Therefore, to find the middle element of a linked list with n elements takes n/2 tests --its O(n).

Therefore, using binary search doesn't help.
