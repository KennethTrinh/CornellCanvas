Author: Andrew Chen (staff)

Vote Count: 0

Someone had to write the equals method for String, but for a class that you write, you have to write the equals method. By default equals is inherited from class Object, and it is equivalent to == (if you don't override the equals method). Using == on objects is simply comparing the memory addresses to see if they are equal, so it tests to see if two objects are literally the same object , not just equal.

Author: Vladia Trinh (staff)

Vote Count: 1

When you are checking for equality of primitive types, you should use ==. In this case, I'm assuming that height is a float or int, so you should use ==. Using .equals would not work because methods can only be called on objects not primitive types. As for your second question, this can be omitted. If you just say height, Java recognizes that you mean this object's height, so you don't have to explicitly say it.
