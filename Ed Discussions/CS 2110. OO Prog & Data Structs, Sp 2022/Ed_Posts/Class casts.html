Author: Andrew Chen (staff)

Vote Count: 1

It fails if Animal is not a subclass of Object. In this case Animal IS a subclass of Object, so it will not fail at runtime. However, if you have two classes that have a parent-child relationship with each other, but one of them has compile-type that is not compatible with the type we're casting to, then you will fail at runtime, for example: class A { } class B { } Object o = new B(); A a = (A) o; Compiles because Object and A, and Object and B have a parent-child relationship. However, B and A do not, and this will fail at runtime with a ClassCastException.
