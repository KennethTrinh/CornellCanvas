Hello, I'm a bit confused about the meaning of the second if statement. Based on the algorithm, if the first if statement isn't true (w is not in S or F), doesn't that mean w has to be in F for the second if statement to be true (d[f] + wgt(f, w) < d[w])? Would it be safe to assume that w is always in F when the code enters that the second if statement block?

YES. The first if-condition is "w is not in S or F". If it is false, then "w IS in S or F". It could be in F, it could be in S. It doesn't matter which. But yes, if d[f] + wgt(f, w) < d[w] that tells you a shorter path to w has been found. That cannot be if w is in S because the shortest path to w is already known. Therefore, if d[f] + wgt(f, w) < d[w], you know that w is in F.
