Yes, Java Swing looks outdated. This is a "feature" of any GUI library that has to be broadly cross-platform, working on any OS. JavaFX does better, but ultimately it's going to be a problem no matter what.

For the best look and feel, one has to work with the GUI libraries available on a particular platform. That introduces significant software engineering challenges. Note, for example, how MS Word is not the same on Mac vs. Windows. Or how LibreOffice is so different from MS.

For games, there are engines optimized to work well across multiple platforms. Consider the Unity Game Engine (e.g. Pokémon Go). According to Wikipedia it now supports 19 platforms. But it is special purpose and has a proprietary license.

For more about games, see CS 3152 whose prereq is CS 2110 — which you have now finished! It uses a Java game engine called LibGDX .
