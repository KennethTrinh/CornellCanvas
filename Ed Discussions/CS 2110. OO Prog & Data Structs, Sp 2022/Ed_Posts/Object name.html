Essentially, yes, the name of an object can be thought of as CLASS_NAME@MEMORY_ADDRESS . Of course, it's just an abstraction we use in CS 2110 along with the "manila folder" for what an object really is - just a specific set of bits in a specific memory address. And note that we distinguish different objects of the same class by denoting them with different memory addresses.
