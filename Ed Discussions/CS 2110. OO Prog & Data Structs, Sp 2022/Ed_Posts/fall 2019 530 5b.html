How would you implement binary search on a doubly linked list? There's no efficient way to know which "half" of the list contains the element you're searching for as with sorted arrays, so you would need to traverse each node. Hence, O(n).

Even if you somehow knew which part of the DLList to start your search from, element lookup time in LinkedLists is not O(1) as with arrays. So you can't "jump" to a particular node here to save time.
