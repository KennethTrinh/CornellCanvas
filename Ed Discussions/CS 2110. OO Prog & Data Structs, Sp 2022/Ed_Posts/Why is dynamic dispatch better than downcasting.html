Suppose we have classes Dog, Cat, Pig, etc. Each is a subclass of Animal. Suppose animal does not have method mm but all the others do. Then, for a variable Animal c; to call a possible method mm in that object you have to write: if (c instanceof Dog) { ((Dog)c).mm(); } else (c instanceof Cat) { ((Cat)c).mm(); } else (c instanceof Pig) { ((Pig)c).mm(); } ... and whenever you add a new subclass (example Bat) you have to add another else-clause. If you don't have OO in your language, then you are almost FORCED to write code similar to the above. However, add method mm to class Animal. Then, instead of that if-else sequence, write simply c.mm(); and the Overriding/Bottom-up rule, which is dynamic dispatch takes over and works.
