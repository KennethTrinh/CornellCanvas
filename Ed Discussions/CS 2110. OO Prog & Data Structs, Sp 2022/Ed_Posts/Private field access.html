If a field is private, can we still access it in the methods? and Can you access it directly if it's another instance of a class (when comparing two instances)? For example if you had a field private int x; And you wanted to use it in a method, could you just use ‘x’ or is a getter necessary. And when comparing, are you able to do (where instance2 is an object of the same class): x == instance2.x;

Private fields are accessible at the class level only. That's class level, not just instance level. So if the method you are describing is a method within the class that contains the private field, then yes, the field is accessible. And for your example with instance2, if instance2 is of the same class as the one you are inside, then x == instance2.x would compile. Again, since private is at the class level, not just instance level.
