I made this question public (and made you anonymous). Please don't make questions private; others may have the same questions. This is important: At compile-time, absolutely nothing is known about what pointer will be in a variable at runtime, and probably that runtime value will change from pointing from one object to another as execution proceeds. Therefore, at compiletime, only the TYPES of expressions are used. Consider (String) k. At compile time, look at it as (String) (an expression of type Integer) It is known at compiletime that an Integer object cannot have a String partition. Therefore, it is a compiletime error Consider (Integer) ob. At compiletime, look at it as (Integer)(an expression of type Object) At runtime, that (expression of type Object) COULD contain a pointer to an object that has an Integer expression! Therefore, that cast is allowed. At runtime, if ob points at an Integer object, fine. But if it points at any other object, A ClassCastException is thrown. Does this help?
