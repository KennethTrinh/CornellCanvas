You're correct that Dress3 wouldn't compile because it doesn't implement getColor().

BUT, the problem then asks which classes will fail to compile even if they are abstract. In this case, if Dress3 were abstract, then it doesn't need to implement getColor(). However, because it contains "new Dress1()", and you cannot instantiate abstract classes, this fails to compile still even if it were abstract.
