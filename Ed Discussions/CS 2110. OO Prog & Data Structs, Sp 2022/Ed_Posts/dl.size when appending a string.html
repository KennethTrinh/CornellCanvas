When DLList is declared a <String>, the method toString() uses the line of code: 

if(n!=null) res.append(", ");

Since the <E> is a String,

", " is also a String,

and there is a method append() in DLList,

How come when appending a second node during the testing of append(), our DLList.size() is 2 and not 3? How come the program doesn't compute ", " as another node? 

1. public String toString() {

2.     var res= new StringBuilder("[");

3.     var n= head;

4.     while (n != null) {

5.       res.append(n.val);

6.        n= n.next;

7.        if (n != null) res.append(", ");

8.     }

9.     return res + "]";

10. }

I understand that res is declared a Stringbuilder, hence in line 7. res.append(", ") means Stringbuilder.append(). But by that reasoning, how come in line 5. res.append(n.val) , where n.val is also a String refers to DLList.append() ? How do I differentiate between which append() is called, if they both have the same call-signature?

res.append(n.val) calls the append() from StringBuilder. You know which method is called if you know the type of the object before the dot. For example in this case, res is before the dot so append is called from the StringBuilder class since res is a StringBuilder object.
