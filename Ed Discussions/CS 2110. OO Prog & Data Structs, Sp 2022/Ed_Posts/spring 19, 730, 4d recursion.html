This is because we already have the recursion with maxPathProduct(child). Also the largestChild can change. If we first find the maxPathProduct of child1, largestChild is set to the path of child1. But the path for child 2 could be bigger, and the largestChild will be reset to path of child 2 etc. We can only be sure the child is the largest when we go over all the children. Thatâ€™s why we do the multiplication after the loop terminate, at the very end in the return statement.
