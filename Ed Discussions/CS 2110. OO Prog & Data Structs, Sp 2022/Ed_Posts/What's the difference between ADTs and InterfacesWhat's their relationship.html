What's the difference between ADTs and Interfaces? What's their relationship?

I know that Array, List, Map, Queue, Set, Stack, Table, Tree, and Vector are ADTs. And class ArrayList implements type List.

So does this mean that List is an ADT and an interface? I'm getting confused by this...

Author: Richard Gu (student)

Vote Count: 1

I believe all ADTs (as a model for useful data types) are interfaces. But interfaces are broader and serve as an abstraction for other non-data type classes that we saw in Prelim 1. So all (or at least almost all) ADTs are interfaces, but only some interfaces are ADTs.

List is an ADT and an interface, since you can't implement List directly and need to instead instantiate ArrayList or LinkedList.

Hope this helps.

When we show you an ADT --abstract data type-- we tell you what the set of values are and its behavior: what operations there are on it. For example:

Stack of integers. Values: all the integers Operations: size, push value onto the stack, pop a value from the stack

This definition has nothing to do with Java! It's just a definition.

It CAN be implemented in Java, but it can be implemented in lots of other language too --Python, Matlab, Pascal, Ruby, Lisp, ...

-------------------------

We can use the Java interface to define this ADT for use in Java, as shown below. Here, the interface declares the operations as methods, giving you information about how to call them. Comments define the behavior --what the methods do. Other languages may have their way of defining this ADT. Many do not.

Then, in Java, one writes a class that implements this ADT.

/** A stack is a bunch of integers, with a LIFO behavior, like a bunch of cafeteria trays */ interface StackOfIntegers { /** The size of this collection */ int size();

/** Push e onto the top of the stack. */ void push(int e);

/** Pop (and return) the value at the top of the stack. */ void pop(); }
