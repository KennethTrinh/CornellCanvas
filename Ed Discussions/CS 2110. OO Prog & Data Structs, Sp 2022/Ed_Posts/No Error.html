A few notes to answer your question: JShell doesn't require us to put semicolons at the end of statements. The end of the statement is determined when you press "return" on your keyboard. As Professor Gries mentioned in #77, "[JShell] is not a complete Java program but just a way to try things out." JShell is not an IDE. It's important not to get confused between JShell and programming in an IDE, such as Eclipse. Semicolons are required if you are programming in Java in an IDE. The semicolon you see in the JShell was extraneous. We didn't need it. But the code will still compile even with extra semicolons at the end of the statement. Again, this is just with JShell, which is not the same as programming in an IDE. To address your question on c3/c4: c3 would be null as well (If class Count had been declared), but that's irrelevant because the Count class itself hasn't even been declared yet, so you are not allowed to reference c3. In fact, this is only allowed in JShell, and the code written would not compile in an IDE (you would get a syntax error "Count cannot be resolved to a type"). The Counter class has been declared so we can declare a variable of type Counter, and we don't get that message. However, by default c4 points to null.
