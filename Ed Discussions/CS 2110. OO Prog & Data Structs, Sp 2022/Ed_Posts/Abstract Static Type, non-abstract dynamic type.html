Are we allowed to declare an object like: Abstract-Class obj1 = Implementation-Class obj2? As an example, Shape Triangle = Triangle t1; Where Triangle is an implementation of abstract class Shape . the dynamic type of Triangle will be Triangle which is an extends the Abstract class Shape (subclass) so you are allowed to do that, also to initialize the variable Triangle you can just do Triangle t1 = new Triangle(…); Shape Triangle = t1; However this limits you do only perform methods from the class Shape when using the variable Triangle, so if the class triangle has new methods that aren’t in class Shape you can’t do Triangle.mm(); assuming mm is a method in class triangle and not shape. This would violate the compile time reference rule
