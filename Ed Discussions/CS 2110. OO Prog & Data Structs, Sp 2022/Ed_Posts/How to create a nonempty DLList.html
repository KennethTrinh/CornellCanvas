Author: Atulya Lohani (student)

Vote Count: 2

If there is only one node in the list then the head and the tail are both the same node. Even if the value of the only node inside the list is null, the list is still nonempty. You can't instantiate a DLList with values already inside it because the constructor doesn't take any arguments. You must use a method that puts a node into the list.

Author: Andrew Chen (staff)

Vote Count: 1

Adding on, the head of the list being null versus the value of the head being null are two completely separate things. The value of a Node can always be null, just as it can be an Integer (1, 3, 5, etc.). The head being null would indicate that the list is empty, meaning the tail would also be null. So the head and tail are either both null or both non-null.
