Title: Eclipse: Failed to init ct.sym for …

Category: General

Vote Count: 0

Problem:

When creating a new project, there is a red "x" on top of the project icon, and Eclipse's "Problems" view shows the error "Failed to init ct.sym for …"

Diagnosis:

Eclipse seems to easily get confused when both a JRE and a JDK are available for the same Java version (many Eclipse problems we've seen this semester relate to a JRE being used instead of a JDK). The solution is to remove Eclipse's knowledge of the JRE.

Solution:

Go to Window>Preferences, then Java>Installed JREs. If there is more than one entry in the list, select any that have "jre" in their name (instead of "jdk") and click the "Remove" button.
