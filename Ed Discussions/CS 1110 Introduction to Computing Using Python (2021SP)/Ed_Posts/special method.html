Maybe the easiest way to think about special methods is as "the common methods we know about with the double-underlines", like __init__ , __str__ , __eq__ .

They are methods that aren't usually called directly, but rather underly calls like when a new object is created, printed, or compared. For example, if we have a class named ExamQuestion, then for calls like

eq= ExamQuestion()

eq2 = ExamQuestion(optional=True)

the righthand sides secretly involve a call to the __init__ for ExamQuestion. A call like str(eq) secretly involves a call to the __str__ for ExamQuestion. An expression like eq == eq2 --- OMG I did not think ahead to have a variable named "eq" and to be talking about the __eq__ method, that's just a massive coincidence --- anyway, an expression like eq == eq2 secretly involves a call to the __eq__ method for ExamQuestion.
