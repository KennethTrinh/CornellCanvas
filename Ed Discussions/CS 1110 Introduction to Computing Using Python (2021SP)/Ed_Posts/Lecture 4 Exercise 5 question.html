No, the "0" comes from the function call at the very end of the file: x = foo( 0 ) The code is actually written "confusingly" to make a pedagogical point; to simplify things, consider instead def one_more(x): print("one_more thinks x is: " + str(x)) return (x+1) result2 = one_more(2) result5 = one_more(5) print(str(result2)) print(str(result5)) The first time one_more() is called, it has input 2, which it stores in its parameter variable x, and then computes with x. The second time it is called, the function has input 5, which it stores in its parameter variable y, and then computer with x.
