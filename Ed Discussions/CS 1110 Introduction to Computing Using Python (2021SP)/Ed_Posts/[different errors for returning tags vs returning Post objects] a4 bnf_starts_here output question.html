First, let's recall what the bnf function is supposed to return. The spec says, "...returns a list of Post objects". So, returning a list of tags (strings) is not what the function should do; it should return a list of Posts (objects). So, changing your code to return lists of Posts, not tags, is definitely progress! Now, checking other Ed Discussion posts about what "Wrongly found ... " means: https://edstem.org/us/courses/3933/discussion/353489 , we see that "Wrongly found ... " means: (a) either you returned not-False when the answer should be False, or, (b) at least your code didn't return False, but it didn't return what it was supposed to do, either. So! Your first error message , "Wrongly found back-and-forth for users A and B at Post 0 for k=3" --- that is because, although there is in fact a bnf at 0 of length 3 for A, B, you were returning tags, not Posts. So that's case (b). Your second error message , "Wrongly found back-and-forth for users A and D at Post 011 for k=2" --- let's see. According to the figure, is there a length-2 A&D bnf at 011? Yes. So, what appears to be happening is that whatever your code is returning for 011, A&D, for k=2, it's not what we're expecting (which would be a list of the Post at 0.1.1 and then the Post at 0.1.1.0. This is case (b) again.
