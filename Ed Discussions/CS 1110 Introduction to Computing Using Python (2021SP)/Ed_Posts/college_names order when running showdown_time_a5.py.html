As an aside, your question made me realize the code as written is using the wrong function for sorting. The spec says the Colleges should be in reverse sorted order of number of applicants. BUT, the code as written... 

    college_list.sort(key=lambda c: len(c.rejected) \
                                + len(c.accepted_not_enrolled) \
                                + len(c.rejected) + \  ### <- copy-paste wasn't updated!  Why counting "rejected" twice?!
                                len(c.waitlisted), 
                  reverse=True)

should clearly be: 

college_list.sort(key=lambda c: len(c.rejected) \
                     + len(c.accepted_not_enrolled) \
                     + len(c.accepted_enrolled) \ ### fixed
                     +  len(c.waitlisted), reverse=True)
