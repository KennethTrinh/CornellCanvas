Most methods we have used (e.g. len() etc.) return something and you can assign what is returned to a variable. The reverse() and sort() methods used in the lab didn't, and I got an error when trying to assign them to a variable, e.g. when I tried to do this: x = output.reverse(), where output is a list.

If you take a look at the Python documentation for lists , you'll see that methods that return something will explicitly say "return" somewhere in the specification. Additionally, sometimes you'll see that methods say they modify the list "in place," which means they just modify the list, but not return it.
