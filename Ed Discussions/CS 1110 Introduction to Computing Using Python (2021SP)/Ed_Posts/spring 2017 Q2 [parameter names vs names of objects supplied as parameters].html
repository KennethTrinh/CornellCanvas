In lines 31-33, the Tower objects are assigned to the variables left, right, and middle (which are not strings). In happens to be a coincidence (likely planned to make the code easier to understand) that the object assigned to the variable left also has the name "left". When plan is called in line 34, Python uses the ID of the the Tower object stored in left (as left not as a string is the Tower object), which so happens to have the name "left" (and so on for right and middle).

If it would help for understanding, lines 31-33 could instead assign the Tower objects to the variables Tower1, Tower2, and Tower3 respectively. In this case, we would call plan(Tower1, Tower3, Tower2, len(Tower2.disks)) (but these objects would still have the names "left", "middle", and "right"). The variable names given are just used for conceptual understanding of what's going on.
