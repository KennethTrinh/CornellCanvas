Following up on Elisabeth's true and correct answer: A metaphor that might be useful: imagine you had "modular" housing, with modules corresponding to rooms you can plug together.

There are lots of different types of room modules, like kitchens and bedrooms and libraries. Now consider a bedroom: while you can sleep in any module, a bedroom is specially designed for sleeping in. So, a script is like a bedroom and a module is like any type of room. Every script is a module, a script is intended to be run directly as Elisabeth says, and every module could be treated as a script ... Just like every bedroom is a room, and a bedroom is intended to be slept in, even though any type of room can be treated as a bedroom if needs must! (Any port in a storm, as they say)

That's a good question. My understanding is that a script is a file that you run directly (say, from a command prompt), while a module is code that gets imported into another file. So the difference is in how you use the file more than its contents.

you can see more details at: https://stackoverflow.com/questions/2996110/what-is-the-difference-between-a-module-and-a-script-in-python
