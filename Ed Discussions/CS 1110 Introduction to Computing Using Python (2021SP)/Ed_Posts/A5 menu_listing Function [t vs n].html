Great observation! First, changing to \n isn't required for the assignment, so you don't need to worry about making it work with \n. Changing the separator to \n in menu_listing breaks the program because the choice of \t is already hardcoded in another part of the code (in showdown_time_a5.py). A quick fix to make it work again would be to change line 87 in showdown_time_a5.py so that it uses \n, not \t, to split the given list m. Here's the relevant part of the code: # Technical: make cnames a list of names from the menu_listing `m` cnames = [] for s in m.split("\t"): # line 87: change to for s in m.split("\n"): cnames.append(s[s.index(' ')+1:]) Now, here's how we could debug this issue. The given error message shows that there is an IndexError at line 111 in showdown_time_a5.py: c1 = sdu.college_named(cnames[int(cnums[0])], colleges). This tells us there must be something wrong with the list (cnames) and/or the index (cnums[0]). So, the next thing we could do is to check the actual values of cnums and cnames using print statements to see what may be causing the IndexError. It turns out cnames has the value ['B', 'A', 'D', 'E'] when using \t in the menu_listing function, but it has the value ['B\n1: A\n2: D\n3: E'] when using \n in the same function. We would then look at how cnames is being created, which is happening at lines 86-88 in showdown_time_a5.py, and find that line 87 assumes that the input string m can be split by \t. If we change the separator from \t to \n in menu_listing, we then need to also change the separator for the split function in showdown_time_a5 accordingly.
