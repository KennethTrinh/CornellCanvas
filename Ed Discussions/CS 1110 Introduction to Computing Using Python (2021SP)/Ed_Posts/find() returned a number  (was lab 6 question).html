Hi! The Python documentation, linked on the course homepage under item 3 "Frequently accessed servers and sites", explains what builtin functions do. Searching there, we find documentation for the string method ffind ( sub ), and importantly, find() returns a special number, -1, if the sought-after substring sub is not in the string str . So, right --- find() does not report an error for non-found substrings, whereas index() does "throw" an error. Keeping find and index() straight --- or, actually, knowing how to find the documentation for them --- is just one of those things Python programmers need to learn to do, just like for learners of French, one has to remember that "table" is feminine whereas "livre" is masculine.
