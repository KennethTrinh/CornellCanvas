We absolutely want you to get in the habit of writing test cases for functions before you write the function bodies. This is one of the new "habits of mind" we referred to in the introductory lecture.

Remember the lesson of Lecture 6, slide 13: " Test Cases can help you find errors in [your understanding of] the specification ".

We know students don't find this natural at first. That's why we've set things up to force this new behavior: the A1 instructions say to do the test cases first, the files are named a1_ first .py and a1_ second. py , and the test cases are due before the function bodies. We promise: our own programming experience has taught us that doing the test cases first really helps you write better code, and write code better!!
