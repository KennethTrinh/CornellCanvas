Hi, I think this reply should help: https://edstem.org/us/courses/3933/discussion/353489?comment=825682 -- wrongly found means either: (a) your code is saying there's a back and forth when there isn't --- not applicable here because there is a k=3 A&B bnf at 0, or (b) it's true that there is such a bnf, but your code is returning the wrong thing --- wrong length, wrong types of items in the list, not a list at all, something like that.
