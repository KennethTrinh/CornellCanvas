There is such a thing as infinite recursion in some functional programming languages like OCaml (see this StackOverflow post) but it's rather complicated -- you won't have to worry about that until CS 3110. (See this link for some info from the online CS 3110 textbook from Spring 2019.) The reason we don't do that in Python is that recursion uses call frames for each function call, which takes up memory proportional to the number of times a function is called. If you try to call a function a large number of times, you'll run out of space!
