map() is cool! (If you're into that kind of thing). But you are not required to use map() and are not allowed to on Prelim 2. ............... Giant footnote begins here! But, so you know: It can, for instance, often serve as a more compact/elegant replacement for the for-loop+recursion pattern. Here's an example, motivated by linguistic analysis, from Spring 2014:

def embed(theinput): """Returns: depth of embedding in theinput. Precondition: theinput is a list of strings or a string. ['hit', 'a', 'guy', 'with', 'glasses'] --> 1 ['hit', ['a', 'guy'], ['with', 'glasses']] --> 2 ['hit', ['a', 'guy', ['with', 'glasses']]] --> 3 ['the', [['red', 'house'], 'and', 'barn', ['that', 'jack', 'built']], 'was', 'razed']: 3 'a' --> 0""" if type(theinput) == str: return 0 else: # the "1" is for theinput being already a list; # now look at the embedding for each item in theinput return 1 + max(map(embed, theinput)) the_inputs_above = [['hit', 'a', 'guy', 'with', 'glasses'], ['hit', ['a', 'guy'], ['with', 'glasses']], ['hit', ['a', 'guy', ['with', 'glasses']]], ['the', [['red', 'house'], 'and', 'barn', ['that', 'jack', 'built']], 'was', 'razed'], 'a' ] for item in the_inputs_above: print("The item is: " + str(item)) print("And embed returned: " + str(embed(item)) + "\n")
