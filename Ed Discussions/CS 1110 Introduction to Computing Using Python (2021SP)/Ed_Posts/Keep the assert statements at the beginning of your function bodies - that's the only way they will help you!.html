Title: Keep the "assert" statements at the *beginning* of your function bodies - that's the only way they will help you!

Author: Prof. Lee (she/they) (admin)

Date: 2021-04-12T05:04:03.913306+10:00

Category: Assignments

Vote Count: 0

Hi all,

We had anticipated some of the issues students would wrestle with, and to fend them off, had added extra assert statements to the function bodies. These help check the types of the arguments.

However, for these assert statements to give you the intended warnings at the right time, you need to obey the instructions: 

# STUDENTS: leave these assert statements here. They help prevent recursion # errors where calls are given unexpected argument types. 

# STUDENTS: Complete the implementation, making effective use of recursion.

That is, make sure you did NOT delete the assert statements, and have those `assert statements be the first lines of your function implementations: start your code where it says "Complete the implementation", not beforehand.
