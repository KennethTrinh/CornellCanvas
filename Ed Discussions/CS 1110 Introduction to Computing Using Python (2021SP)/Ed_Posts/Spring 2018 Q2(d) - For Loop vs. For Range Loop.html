This goes back to that whenever you are editing something in an iterable, you want to use a range loop. In this case, when you redefine item in line 8, you are re-defining the loop variable item, not the element in the_menu (as such, this change will be lost as soon as you loop away from that item, and won't have any lasting impact on the_menu)

(Sometimes it won't matter, such as if you're editing only attributes of what you're looping over, and not the actual items themselves. However, it's usually safer to loop over the range - conceptually, it's a bad idea to loop over something that you're editing at the same time)
