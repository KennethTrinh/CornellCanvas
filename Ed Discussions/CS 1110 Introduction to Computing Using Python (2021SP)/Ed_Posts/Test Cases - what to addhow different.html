You are free to add to them, but we are looking more for “test coverage,” which is that the tests you make cover “more” or prove more the correctness of your code. That is, make test cases that cover other aspects of your code being correct, more specifically edge cases or interesting/special inputs that align with the precondition and that maybe you wouldn’t think about when actually coding it up (inputs that work but aren’t what one would say “normal”).
