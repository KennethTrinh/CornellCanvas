Author: Prof. Lee (she/they) (admin)

Vote Count: 1

Good question! This is a bit complicated, and there are various ways to answer it. (a) I think for the purposes of CS1110, maybe the best way to think of this is that indeed, double-underscore methods are exceptions to the usual convention/preference. [footnote 1] (b) What was said about "_score" was actually on a different topic. "_score(...)" has only a single underscore in front of it, actually [footnote 2]. The note was meant to indicate: you can think about whether to leave _score(...) as is, or to redefine it as method with name score(...) (with no underscore). [footnote 1: special, or double-underscore methods, are even processed differently by Python, in that they work on the class, not the object level, for Reasons. If you are interested, see https://docs.python.org/3/reference/datamodel.html#special-method-lookup but please don't feel that you should be able to understand what that document is talking about --- especially since CS1110 is an intro to programming , not to the particularities of Python per se. (alliteration, hurrah)] [If you're typographically inclined, you may find the different uses of underscores in Python semi-interesting. See this part of the Python style guide: https://www.python.org/dev/peps/pep-0008/#id36 ]
