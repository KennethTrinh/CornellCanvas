It's unclear what you mean by a "base case where I check if a list value is equivalent to v," but I suspect this is where your issue lies. Often for simple recursions on lists, we want to check the 0th value in the list and then all the remaining values (i.e., recurse on the list w/ the 0th element removed). It may help to ask yourself the following question: "Given no further information, what do I know about the number of v's in the list based on what the 0th element in the list is?"

Another hint, you don't necessarily need a base case other than if the list is empty, but this is ultimately up to how you structure your code.
