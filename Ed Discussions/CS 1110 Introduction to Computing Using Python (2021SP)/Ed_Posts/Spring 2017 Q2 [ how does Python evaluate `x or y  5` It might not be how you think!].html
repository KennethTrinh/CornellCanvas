Ah! I understand your question: A or B < 5 is processed as (A) or (B < 5) not as ( A < 5) or (B < 5) So, since isEmpty is True (in the move() call frame), the condition "isEmpty or ..." short-circuits to True (Python doesn't have to evaluate topDisk()), and so line 16 is entered.
