Essentially, I like to think of our 2^12 as the amount of different ways we can draw arrows from a vector of 12 things to a vector of 2, where this describes all of the possible functions we can draw between two sets. For each of these arrows corresponds to an input, and for each input, we multiply the amount of hypothesis we can have by 2 since there exists a subset of all hypotheses where that label is 0 and that label is 1 (to discretize the label into a binary number). Now, for those 5 inputs, we cannot draw two arrows we can only draw one, therefore, since they can only provide one mapping, the size of our version space is from how many different mappings we can do for the inputs which don't matter. Hope that helps :-)
