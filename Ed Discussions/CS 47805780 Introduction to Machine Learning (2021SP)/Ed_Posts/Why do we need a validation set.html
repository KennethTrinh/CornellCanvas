Maybe I'm missing something obvious, but why do we need a validation set? The slide below explains that we use $S_{val}$ to evaluate how well the parameters of our learning algorithm perform on the data. But why can't we just use $S_{train}$ to do this? (I.e. train the algorithm with different parameters on $S_{train}$ and just choose the one that achieves the lowest error.)

Author: Victor Butoi (staff)

Vote Count: 1

Essentially, because what you care about with model performance isn't how well you can optimize your ML algorithm to fit a set of data, but how well you can create a model that will generalize to a real world distribution.

If you only look at the training set and attempt to tune your hyper-parameters from that; you have no idea how those hyper-parameters will make your model overfit your training set as well as how it affects your ability to generalize. The validation set then allows for this intermediate check of your model's performance on data that it isn't trained on so that you can choose the hyper-parameters which allow for the validation to be as low as possible. Now, you might say that what if we overfit our validation set? This is 100% a real issue that happens; however, it then becomes a bit weird what the word overfitting means. Since if the validation set is a representative sample of our the real-world distribution looks like, then overfitting it just means that we will perform well even on data that our model hasn't corrected itself with.
