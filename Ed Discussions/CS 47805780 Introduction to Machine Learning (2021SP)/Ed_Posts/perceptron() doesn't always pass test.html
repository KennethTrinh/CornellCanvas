I think it's ok if it doesn't always pass the test that's in the cell following your implementation. However, it should always pass the tests that are graded because of the fact that the total number of updates that perceptron has to perform is bounded above by the maximum radius squared over the margin squared. So, I'm assuming for those tests the data sets picked have a known radius and margin so that the max number of iterations chosen will always guarantee convergence.
