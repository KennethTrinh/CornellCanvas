Author: Thorsten Joachims (admin)

Vote Count: 1

There is actually quite some flexibility in how you implement the Perceptron algorithm, since the upper bound on the number of updates holds for any order of examples. So, you could repeatedly update on the same one until you get it right, always pick a random one, always go round-robin, or any other order. We will learn in a few lectures that always picking the next example uniformly at random actually has some attractive properties as a "stochastic gradient descent" procedure (will explain in a few lectures what that means).

For the homework, please stick to the algorithm on the slides.
