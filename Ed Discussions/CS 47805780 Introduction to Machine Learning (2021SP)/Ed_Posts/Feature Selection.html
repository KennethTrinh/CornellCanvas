Kind of related to #1416 but I was wondering if forward or backward selection would ever account for the best subset of features. Like for instance you had feature vector that was [age, gender, income, education]. With forward and backward selection you would always go through the same order, right? If that's the case, what if it turns out that age and gender don't work so great together, but gender and education produce a better model? With forward and backward selection, wouldn't gender be dropped, so you wouldn't be able to test gender and education together?

I think this post does a good job talking about all of the different types of subset selections (though the code is for R). Basically you are right, this doesn't consider all combinations. Doing that can be taxing for larger datasets. There are options for considering all combinations, you just have to be careful with the number of features you have (https://daviddalpiaz.github.io/appliedstats/variable-selection-and-model-building.html).
