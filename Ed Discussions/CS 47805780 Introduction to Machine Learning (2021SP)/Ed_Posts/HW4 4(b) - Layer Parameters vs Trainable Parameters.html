Author: Jane Du (staff)

Layer Parameters column: just the different specs you need to understand a layer. For example, you can see that Conv 2 has a filter of side length 15. Trainable parameters column: for each row in the table, write down a single number that is the number of trainable parameters in that layer. You should know how to compute this depending on the type of layer (FC, Conv, etc.) At the end of the problem, provide a total count of trainable parameters -- the sum of the trainable parameters column.

Author: Jane Du (staff)

Vote Count: 1

This refers to the specs of the conv layer. You expect to see this expression in the column "layer parameters". Using these numbers, you should be able to calculate the number of trainable parameters from this convolution layer. Put that count into the last column. You should be able to do this for every layer, of all the types! Some of the layers, like "flatten", don't have any specs -- it's just a hardcoded operation. And, some of the layers don't necessarily have any trainable parameters (i.e. weights).
