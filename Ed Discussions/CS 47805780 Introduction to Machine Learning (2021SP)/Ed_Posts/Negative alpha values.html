In my dualqp() implementation, cvxpy does not seem to be adhering to my alpha >= 0 constraint. My implementation passes the test cases, but when I print the alpha values, they are half negative and half positive. They are all very small values, hovering very very close to zero (e-10 or e-11). If i change my constraint to be greater than a different number, say 5 or 0.1, it does not violate the constraint. It only appears to violate the constraint when forcing alpha >= 0 . I did try alpha >= 1e-99 , which also failed, giving negative alpha values. However, the function passes its test cases with alpha >= 0 .
