Think about it this way. For each output map , there is an fxf filter applied to each of the input maps (which then get added together to produce the output). So for instance, let us say we are looking at a 512x512x3 image, with a convolution layer with 64 output filters (3x3). This means, that each output map has a 3x3 filter for each input channel (of which there are 3); each filter has different 3x3 (they're not identical). This implies there are 3x3x3 weights per output map. But because, the convolution layer has 64 output maps, this is done 64 times with different weights/filters.

64 is a number decided by the person designing the architecture. When I define a convolutional layer, I have to define (1) the number of filters (in this example it's 64), (2) filter size, (3) various hyperparameters like padding/stride/etc. In the example that you posted (image), the number of channels in is 3, the number of output filters is 2, and the filter size is 3x3
