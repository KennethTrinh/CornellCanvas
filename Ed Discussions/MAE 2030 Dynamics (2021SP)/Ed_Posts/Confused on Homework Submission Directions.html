A README file is a file named, appropriately, "README". It could be plain text or pdf. It has text in it that tells the reader what is in the other files in the directory. It's like a table of contents, or index, for a collection of files in a folder. When a reader opens a folder the first thing they do, or should do, is read the README file. The most clear thing is to have your pdf be complete in itself, with no need for a reader to look at anything else. Thus it should include, if nothing else, at least as an appendix ,any code you wrote and used. But, you should also be handing in functioning matlab code so your peer graders or real grader can try it and see how it works.
