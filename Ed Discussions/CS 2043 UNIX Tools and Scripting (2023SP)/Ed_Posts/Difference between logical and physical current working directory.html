I can show you the difference. I created a test folder called test , located in ~/test/ , I then create a symbolic link in my home directory to my test folder called asdf , it is also located in ~/asdf . Let's change our directory to follow the symbolic link: cd ~/asdf If you do pwd -L , it shows us /home/[username]/asdf , this is our "logical" working directory. If you do pwd -P it shows us /home/[username]/test , this is our "physical" working directory, because this is the actual directory we are working in after you remove the symbolic link that links asdf -> test I recommend you try this yourself (In your VM with a test directory) so you can see this for yourself.
