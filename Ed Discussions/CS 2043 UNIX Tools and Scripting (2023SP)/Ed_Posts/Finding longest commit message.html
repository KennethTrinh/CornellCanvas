All, I just posted this answer to a private question on Ed, posting it publicly here for all to benefit from: You may assume that the commit message is contained on exactly one line. In fact, if you just use "git log" as your source of information (and not additional switches that may limit the output), you will find that git returns exactly 6 lines for each commit: commit [hash information] [branch information] Author: Date: blank line commit message blank line which means if you are using process substitution to take the output of git log and send it to a for loop that is reading lines, it's reasonable to expect that you'll have the entire commit message in some sort of shell variable. Once you have the commit message in a variable, you can use the following special syntax to get the number of characters in the commit message (and we should use number of characters and not number of words to determine which commit message is the longest): Below... if I have a string in a variable named "somevar", I can get the length of the string as follows: 
ron@Sandcrater-MBAir-2 ~ % somevar="This is a test" 
ron@Sandcrater-MBAir-2 ~ % echo ${#somevar} 
14
