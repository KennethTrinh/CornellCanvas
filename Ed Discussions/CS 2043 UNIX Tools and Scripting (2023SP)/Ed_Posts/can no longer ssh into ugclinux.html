Try using the DNS name to ssh into it. As you may have already determined, ugclinux is a cluster of separate machines to help balance the load of all the students using it. Each separate machine has its own IP address. When you use "ugclinux.cs.cornell.edu" with ssh, that name might resolve to any of the IP addresses used for each of the physical machines the make up the ugclinux cluster. If when using the IP addresses directly that particular machine becomes unavailable or it's IP address changes, you can see the behavior shown by the screenshot you posted. Regardless of which physical machine you end up connecting to, your home directory will be accessible as it is shared across all machines. However, the /tmp directory is not which is why the homework instructed you to ssh into a particular IP address so that you can retrieve the .zip file to start the assignment. Let me know if ssh'ing into ugclinux.cs.cornell.edu doesn't work for you!
