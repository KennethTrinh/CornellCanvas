Not really. Without the dot notation, you're specifying an absolute path. This means that your script won't work unless the required files (nouns, sentence.txt, etc.) are located in the VM's base directory (/). By adding the dot notation, you are indicating that the path is relative , meaning that the starting point of whatever filename or directory following the dot is the current directory that the script is being run. The relative path notation makes more sense here since the script should work by running ./a2.sh whenever the required files or directories are in the same directory as the script. So a) make sure you don't have all your files in the base directory ( /A2/adverbs/... , etc.) but somewhere else ( ~/A2/adverbs/... for example), and b) you're running the script as ./a2.sh on ~/A2 ).
