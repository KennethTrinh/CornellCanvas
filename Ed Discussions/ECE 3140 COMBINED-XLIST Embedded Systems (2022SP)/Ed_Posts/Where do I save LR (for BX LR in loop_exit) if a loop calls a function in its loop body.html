Do I push LR on the stack before I call the loop? Yes, because otherwise as you said you would push it in for each iteration of the loop. Is BL to a loop a caller-callee structure? Or is that only for functions? Does caller-callee apply to recursive functions (they are almost similar to loops)? You should only be using the BL instruction for subroutines since it branches and links (i.e. it jumps to that point for execution and moves the address of the instruction after the BL instruction into the LR). The ARM calling convention applies to subroutines, but that being said if you had a loop within a subroutine that overwrote registers it wasn't supposed to then that could cause your subroutine to not follow the ARM calling convention Does caller-callee apply to recursive functions (they are almost similar to loops)?
