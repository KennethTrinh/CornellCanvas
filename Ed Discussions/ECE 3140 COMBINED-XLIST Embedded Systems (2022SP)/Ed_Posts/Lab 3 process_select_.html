Hi, I was wondering if when cursp is NULL in function process_select, does that mean that we should return NULL always, or is that when we need to take other action to select a process to run?

Author: Harris Miller (None)

Vote Count: 0

If cursp is NULL , you need to take appropriate action on the current_process (if such one exists).

If there is no process ready, process_select should return NULL . If there is a ready process, process_select must return the value of the stack pointer for the next ready process.
