Since it's arbitrary, can we store returned values in the stack?

------------------------------------

Author: Nils Napp (admin)

Vote Count: 0

No. You cannot return things via the stack. It has to follow the calling convention: The SP has to be in the same location before and after the function call. The only possible return value should be in R0. (A technical exception is if you were to pass the address of a local variable into the function and use it store the result there). Then, the computed value is technically stored on the stack (because it is a local variable), but it is not accessed via push/pop, and the SP is still in the same location before and after the function call.
