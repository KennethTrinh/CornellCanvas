I'm currently struggling with the difference between POP{LR}, POP{PC} and BX LR. From my understanding, all 3 functions return to where the branch link occurred. However, not all of them were interchangeable when I was working on Lab 1 so I was wondering what the difference between the 3 was and how we know which one to use.

Push/Pop {register} stores the address in the stack and manipulates the stack pointer, where as BX branches to the address specified in LR. If you are calling just one function inside your main, they both could work. Imagine, however, that you have a recursive function call, LR would we overwritten every time you go one layer deeper so your system won't remember the earlier return addresses. To expand on the answer. POP{LR} does not actually return from a function call. (It also does not compile on ARMv6-M!). Conceptually, you could do: POP{LR} <--loads return address into LR BX LR <-- actually returns by loading LR into PC You can do this in one step via: POP{PC} If you wanted to the two-step route were you first pop the return address into a reg and the Branch, you need to use a low register on ARMv6-M. So something like: POP{R3} BX R3 The value in R3 originally came from LR when it was pushed, but you can put the value into any register you want. Between the different options, POP{PC} is the easiest.
