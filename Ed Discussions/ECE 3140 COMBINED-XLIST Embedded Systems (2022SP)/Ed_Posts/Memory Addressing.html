When accessing memory, we use a 32-bit address to specify the byte. But, doesn't a typical load instruction access all 32 bits of the word? In other words, if we are only specifying the address of the byte, how do we get the other 3 bytes in the word? I assume how this is done is we specify a base address and it is implied that we want the entire word? If this is the case, does the base address depend on whether it is little or big endian?

When accessing memory we need to specify which address we want read data from in memory. To specify a memory address we need 32 bits since we're using 32 bit addressable memory. For the load instruction we provide the memory address of the byte of our data at the lowest address. To answer your question explicitly: this is independent of whether the system is using big endian or little endian. In our case, since we're using little endian the base address happens to also be the address of the least significant byte of the data that we want. Now, we have several different load instructions that we can use in Thumb ARM. We have LDR which allows us to load a whole word (i.e. 32 bits) at once into a register and this is the one we'll use most commonly. Author: Aaron Wilhelm (admin)

Vote Count: 0

Yes, it does depend on the endianness. The Application Interrupt and Reset Control Register (AIRCR) states the endianness of the system so the bytes are automatically loaded in properly assuming you're accessing memory that is the same endianness as the processor. If not, there are load instructions that reverse the byte order that you can use.
