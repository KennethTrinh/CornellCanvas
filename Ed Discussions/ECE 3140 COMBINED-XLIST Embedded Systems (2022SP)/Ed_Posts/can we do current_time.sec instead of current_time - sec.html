Yes, you actually have to do current_time.sec since current_time is not a pointer to a struct with a sec property. It is just the struct itself with a sec property. pointer_name -> property is shorthand for (*pointer_name).property so you only really need to use -> when you are trying to access the property of a struct that a pointer is pointing to.
