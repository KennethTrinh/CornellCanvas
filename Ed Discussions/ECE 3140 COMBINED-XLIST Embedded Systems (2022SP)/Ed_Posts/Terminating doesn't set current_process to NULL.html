I understand that the lab3 document is slightly confusing. I did not write it but I will attempt to clarify. The support code 3140.s and 3140_concur.c has no reference or notion of process_t* current_process .
This variable is set to NULL when a process terminates, and should also be NULL until process_start() is called.
This should be interpreted as "you should set it [the process pointed to] to NULL when a process terminates, and [the pointer current_process ] should be set to NULL until process_start() is called." Obviously, the current process consists of various bits of malloc 'd data so you will need to deal with that appropriately to avoid memory leaks.
