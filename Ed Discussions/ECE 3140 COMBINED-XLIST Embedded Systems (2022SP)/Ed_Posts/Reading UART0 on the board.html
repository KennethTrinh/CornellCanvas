So here is the best solution I could come up with: We only ever write to the UART when we are done with the game, so we can leverage the receive data register to do this. When we reach game over, Python sends a number on the UART. This will set the receive data register to 1, and we can detect it in the C code with this: 
UART0->S1 & UART_S1_RDRF_MASK
Which is 1 when there is something to receive, and 0 otherwise. Whenever this becomes 1, we know to stop the game. We don't ever read the actual value, although I don't think that matters because we only need the first signal.
