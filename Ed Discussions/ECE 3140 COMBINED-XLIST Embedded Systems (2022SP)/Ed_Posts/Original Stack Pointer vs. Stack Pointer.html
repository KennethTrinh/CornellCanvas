My lab partner and I defined our process_state to have a stack pointer, as well as an original stack pointer, as suggested in the discussion slides. When writing process_create, I was wondering what differentiates the original stack pointer from the normal stack pointer. I know calling process_stack_init returns a pointer to a stack, but we assumed that this was the pointer to the stack of the process that was just created. How do we initialize the original stack pointer without accessing any register values?

You are correct, process_stack_init returns a pointer to the stack of the newly created process and will also be the original stack pointer. Your process will need to have references to both or else it will be extremely difficult to call process_stack_free (see the description above the function definition). Let's assume the condition that you have a single process running. After you execute a single time slot and get sent to process_select following an IRQ, if your process's stack updated (ex: created a local variable), the cursp will now differ from the original stack pointer. If you just returned the original stack pointer and the processor tried to resume context, you are going to have a bad time as you are no longer pointed at the correct point in the stack.
