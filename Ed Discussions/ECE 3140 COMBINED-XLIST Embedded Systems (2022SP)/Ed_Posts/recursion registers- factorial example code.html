The code follows the ARM convention and is passing the first (and only) argument in R0. R1 is used to hold the original value of R0 (i.e. the function argument) so we can use it later to multiply (input argument) * factorial(input - 1). Notice that the code never expects a certain value to be in R1 when the subroutine is called; the first instruction with R1 moves the value of R0 into R1. R1 is pushed and popped before/after the recursive subroutine call since factorial1 is caller saved
