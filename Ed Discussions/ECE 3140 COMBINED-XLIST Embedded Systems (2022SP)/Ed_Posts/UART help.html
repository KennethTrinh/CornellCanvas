EDIT: Serial example has been posted to Canvas. First, it may help if you get your UART code working when hooked up just between the KL46Z and the computer. Luckily, the OpenSDA USB port is connected to PTA1 and PTA2 which can be configured for UART0. (Shown in the FRDM-KL46Z user manual) Next, you want to check out the handy table in the processor manual on page 171 that tells you the code for setting those pins to UART0 TX and UART0 RX (remember all that info from prior labs for setting up the LEDs, also the macro PORT_PCR_MUX(n) may come in handy) To get PuTTY to work, you need to have the correct COM port and baud rate select. In this example, I select "Serial", type in COM5, and a baud rate of 115200. (The most "common" baud rate is 9600 but most modern devices will support baud rates up to 115200) If you do not know which COM port to use, you can open up the Device Manager (on Windows) and look under Ports. The OpenSDA / PEMicro device is the correct one. Chapter 39 (UART0) is a beast to understand so I will see if we will be providing any UART support code this year.
