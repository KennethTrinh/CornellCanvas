Author: Nils Napp (admin)

Vote Count: 0

In both PUSH and POP There are only 8+1 bits to specify the registers in a one-hot encoding scheme, see: A6.7.49 POP (p. A6-165) in the ARMv6-M reference. So you can freely use any subset of R0-R7 in both instructions. The extra bit is used in PUSH to specify if LR should be pushed, and in POP if PC should be popped. This asymmetric behavior is because of how function calls are implemented. You need to PUSH LR all the time, and the only reason to POP LR is to use BX LR, but since BX LR essentially just moves LR into the PC, you get equivalent behavior by using POP {PC}. If for some reason to need to modify the value that was pushed onto the stack from LR before returning (and I really can't think of a reason why you'd ever want to), you could always use a low register, e.g. R2, and do POP {R2} BX R2. LR is not special except that BL stores the return address there.
