Frist, just for folks watching this thread, the BS question does not really relate to the value of the PC but what mux input should be used to select the right MP value. Secondly, I also find this diagram somewhat confusing (for the exact reason you point out)! And the best way to think of it is that how the value of 'OFF' is computed is not shown, and it should both correct for the fact that the operands are in instructions (as opposed to bytes) and that 'OFF' is added to (PC+2) instead of PC. As a minor picking point SE(-1,0) = -1 as opposed to -2. S igned E xtend just changes a bit encoding, so that a negative 8-bit int corresponds to the correct 16-bit int. Otherwise, the 8-bit -1 would be interpreted as a +255 16-bit number. Any suggestions for the update formula for 'OFF' ?

EDIT: So I don't think my previous answer is quite right. I was reading SE() as function application with two arguments. However, sign extending to 0 bits makes no sense. In that case -1,0 should be read as concatenation, so that the added zero is essentially a multiply by two. Then the original post SE(-1,0) is indeed -2. But I think it should read SE(-1:0, 16), i.e. LHS by 1 and then extend to the data width of the address space. You still need to undo the PC increment, though. Sorry for the confusion.
