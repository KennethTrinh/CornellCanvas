Author: Aidan McNay (student)

Vote Count: 0

1) Each lock should have its own blocked queue. It wouldn't make sense to have a global blocked queue, as you want to have a process be blocked only for a specific lock (otherwise, when a lock is unlocked, how would you know which processes to add back to the queue?). Therefore, it shouldn't be part of your process.c file, but somewhere with your lock files. If a process is blocked, it should be added to the blocked queue of the particular lock 2) process_select doesn't add anything to a blocked queue (as it doesn't know which lock it was blocked for). Additionally, where are you getting "block" from in order to check it? To do that, you may have to edit your process_t struct

Author: Aidan McNay (student)

Vote Count: 0

You should block a process (and add it to the corresponding blocked process queue) when the process attempts to lock the lock (but realizes that it's already locked). process_select should then just schedule the next (unblocked) process. To do this, you shouldn't have any blocked processes in your process queue
