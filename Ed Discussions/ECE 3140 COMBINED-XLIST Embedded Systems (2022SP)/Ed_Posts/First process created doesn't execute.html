We had this issue too! In our case, the issue was as follows: Our process_select checked whether a process was currently running (by checking cursp). If cursp was NULL, we interpreted this as the process that was at the front of the queue had terminated, and free'd it appropriately. The issue with this was that cursp begins as NULL, so the first time you call process_select, it will see cursp as NULL and free the first process. You need to figure out some way to differentiate this first call to process_select from all others where cursp is NULL
