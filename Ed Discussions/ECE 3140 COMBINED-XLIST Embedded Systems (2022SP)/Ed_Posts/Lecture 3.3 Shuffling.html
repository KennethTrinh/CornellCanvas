By "shuffling" I mean changing the location of bits in the word. So lets say I read a 32-bit memory line and the bus has the value: 0b 0000 0000 0000 1111 0000 0000 0000 0000 or in hex 0x 00 0F 0000 The layout in memory would be Byte3, Byte2 ,Byte1,Byte0. Where Byte2 = 0x0F (aka 15) and Byte3,Byte1, and Byte0 are 0x00 If I wanted to read Byte 2 with LDRB, then I would need to read the word line (0x00 0F 0000) and then "shuffle" or move the bold bits to the least significant byte. It means that for some types of memory access (the ones that are allowed by the ISA) I can read a single 32-bit line and then get the answer by just moving bits around. But for memory access that is not aligned, I would need to read two consecutive 32-bit words and get bits for each, i.e. you can't get the result by just shuffling/moving bits from a single 32-bit read instruction.
