    Yup! Imagine if a program terminates one line after it starts. Would that affect your performance with our scheduling?

    For new test cases, I think the best thing might be to view each process as a block (with length corresponding to the length it takes), and think of all the ways you might be able to change the picture of scheduling based on that. What happens if we add more blocks? Change the length of some blocks (perhaps differently relative to other blocks?) Another more abstract thing to think about might be block combination (what if we put multiple processes inside a block?) This visualization might be helpful for thinking about all the ways your scheduling setup might change
