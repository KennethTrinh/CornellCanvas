When we allocate memory, we allocate it in the heap space. The stack space is used for stack frames (such as regular function calls). There is a subtle difference between them. If you were to declare a variable in a program regularly, it would be stored on the stack. Often, however, we wish to dynamically allocate space in memory, which comes from the heap space. To help, look at the code below: 
void main()
{
    int a = 1;
    int* b = (int*)malloc(sizeof(int));
}
The variable a would be stored on the stack (as it is declared during a function, main). Malloc would allocate space in the heap (specifically, the amount required to store an int on whatever machine you're using), and would return a pointer to that memory to b. The variable b would live on the stack, but it would contain a pointer to memory on the heap
