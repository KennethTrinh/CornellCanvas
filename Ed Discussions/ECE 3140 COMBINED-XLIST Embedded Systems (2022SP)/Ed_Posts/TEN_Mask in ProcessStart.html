Looking at the assembly, it looks as though this will only be true if you enable timing in process_select. The assembly only stores the current value of TCTRL and restores it later. That being said, you should be able to start the timer in process_start and have it work fine - the process timer should be sufficiently long enough that the time spent starting the first process is significantly small in comparison, so when the interrupt comes from the timer, we will be in the process, not the setup
