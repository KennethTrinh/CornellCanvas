Title: Have your component resize itself as page width changes

Hope it'll be helpful to more than one of you :) To have some component resize itself based on page width, you need some way to keep track of the page width as it changes. To do so, we use something called eventListener . You can read about its documentation here, but I'll outline what you'll need for this task below. 1. In the <script> section of the component (not <script setup > !), add / find the section export default { ... } , and add the following pieces in it: 
export default {
    data: function() {
        return {
            windowHeight: window.innerHeight,
            windowWidth: window.innerWidth,
        }
    },
    emits: ["add-if-you-need"],
    methods: {
        // add if you need
    },
    mounted() {
        window.addEventListener('resize', () => {
            this.windowHeight = window.innerHeight;
            this.windowWidth = window.innerWidth;
        })
    },
    beforeDestroy() {
        window.removeEventListener('resize', () => {
            this.windowHeight = window.innerHeight;
            this.windowWidth = window.innerWidth;
        })
    }
}
We use the data() function to keep track of page height and width, feel free to give them different names if you'd like. The eventListener we add to the window will update those variables as the window resize . Note that we will need to remove these eventListener before we close the window (aka destroy the component) - otherwise you are only adding more and more of them every time you open the page which is not fun to your processor :( 2. You will then be able to use v-bind as you do with other components to "bind" the windowWidth to a component's style, for example: 
<div :style="{backgroundColor: windowWidth > 900 ? 'red' : 'blue'}" > 123 </div>
I believe the above is called "inline styling", which is basically saying the styles go "inline" with your actually component. This can get messy quickly if you do that to all components on your page, and unfortunately there's no good way to use windowWidth or other computed variables in your <style> section. So if you hate inline styling, you might have to stick with: 
<style scoped> // always put scoped here!!
.div {
    background-color: blue;
}

@media (min-width: 800px) {
    .div {
        background-color: red;
    }
}
</style>
