Should the processing dot map have a delay? The drawer responds to the changes in the environment with a significant delay that gets worse the longer the program is running. The dots, however, are being generated at a consistent pace. I have a suspicion that this is because the inputs are being generated too quickly and the program processes each input too slowly. Are we expected to account for this in our code? Or can our project have a delayed response?

The Arduino program on ESP32 should only send data to Processing code when being asked to. The Processing program will send 'N' as the instruction to request data from ESP32. If the program on ESP32 keeps sending data to Processing without waiting for the instruction('N'), it can cause delay, as the visualization program can not process data fast enough.
