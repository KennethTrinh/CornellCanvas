My project explores connections between natural language and source code in technical forums, i.e. StackOverflow. Up to now, my focus has been on obtaining the data, processing the data, and graphing preliminary data from it such as answer score vs. user reputation, answer score vs. post views, answer score vs. user upvotes, etc. Now, I will shift my focus to cleaning the data and determining what subset of the data I would like to conduct my experiments on. Next week, I commit to completing the following: Select a tag category on StackOverflow on which to conduct my experiments. Choosing a tag category will involve also finding a code lexer for the tag language and reading prior works analyzing differences in language of different language communities on StackOverflow. Reformat my data processing code to: extract the answers of ~1000 questions that have many answers. (Motivation is that the shared question can serve as a control between good and bad answers.) toss out questions that have relatively low view count or are relatively new. Also determine what is low view count and what is new. only use answers from low-reputation users. Also determine what low-reputation is. (Motivation is that restricting the range of user reputation can serve as a control, and also less experienced users are more likely to give a range of answer "quality.") draw a distinction between in-text code words like this and full snippets of code. I imagine that while this week is focused on data cleaning with an objective to run experiments once I have the resulting data, I will likely need to come back and re-process the data as I stumble across roadblocks or deeper questions in my experiments. Therefore, it would suit me well to document and code this part well in the upcoming week.
