A couple of clarifications: We don't assume only one initial starter - the problem qualifies this by saying "with a sufficient number of initial users". Therefore, in the cluster that it starts, you can assume enough initial users that the feature/cheat code spreads It means that no matter what the spread, nobody in Y will adopt it (that would be a valid conclusion to come to based on that, though) I don't think we need to ever consider this - we're really only concerned with whether the spread can get from X to Y or vice versa, so we don't need to worry about the spread possibly going back and forth between clusters. We know it will always spread across the cluster it starts in, but are concerned with whether it spreads to the other cluster Hope this helps!
