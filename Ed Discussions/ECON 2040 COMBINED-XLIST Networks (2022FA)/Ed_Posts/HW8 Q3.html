Nope; q=b/(a+b) is a specific case of what's generalized above.

You know that Player u should choose A if the payoff for A is greater than B. You can reduce this inequality to something of the form x>c , indicating that we should choose A if x is above some value c. In this case, this is exactly what we described before with the threshold rule, with c serving the role of q

(Our calculations result in a range of neighbors for u because u will choose A for the range of possible neighbors such that x>c )
