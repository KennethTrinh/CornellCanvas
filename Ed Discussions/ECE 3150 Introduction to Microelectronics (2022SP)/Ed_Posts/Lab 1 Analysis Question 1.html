Alternatively, a code-y solution! Run the following commands on the command line from the directory that you have the file in: 

grep "+" your_data.txt | cut -f1 | tee -a first_column.txt
grep "+" your_data.txt | cut -f2 | tee -a second_column.txt

This will create .txt files first_column and second_column with the appropriate data!

If you're curious, how it works:

  • grep gets all the lines from a file. Here, I'm specifying only get the lines that start with "+" - this will therefore only get the lines with data, not the units and everything at the top

  • cut breaks apart the line - here, using the default delimeter (tab). For the first column, you want the first portion of this data (-f1), and for the second column, you wants the second portion (-f2)

  • tee takes this result and stores it in a text file

  • The | between all the commands pipelines the data from one to the next, so for all the lines that grep gets, they will be cut, and the result from cut will be stored in the .txt file. Pretty neat!

    You can also import it into excel instead of copying it in. Go to File -> import -> select the file type (in general you'll usually see either .txt text files or .csv for specifically comma-separated data files), press import, browse to select your file, then select "Delimited" instead of "Fixed", finally choose the type of delimiter (most of the time, data files are delimited by either a space, a tab, or a comma), press Done, then Ok to place the data in the sheet starting at a certain row and column. Hope that helps!
