This question may not be in scope since it depends on hashing. Basically, counts.put(s, in + 1) usually takes constant time, so since it's done n times the function is O(n). But, in the worse case you have to resize the hash array every single time you insert an element, which means rehashing every single key in the hash map. This takes linear time, so when performed n times, it is O(n^2).
