Hi, for Account b = new InterestAccount(); I know that the static type of b is Account, but what is b's dynamic type when a method is called, such as b.printStatement()? I think it should also be Account, because on the lecture slide for Compile-time vs. Runtime the pointer still points to the Account folder. So it's possible for b to access the Interest Account methods that also exist in Account, even if its dynamic type is Account? Further, is it possible to change the dynamic type? Can b ever have a dynamic type of InterestAccount?

The dynamic type is the type of the object currently pointed to by b. It can change with each assignment to b. In this case, after execution of the assignment, the dynamic type of the object pointed to by b is InterestAccount.

Author: Curran Muhlberger (admin)

Vote Count: 0

When we draw variables as boxes pointing to folders on the lecture slides, we may occasionally draw an arrow pointing to an intermediate partition, as a clue to reflect the static type of the variable. However, because Java uses dynamic dispatch , there is really no such thing as a variable pointing to an intermediate partition. In Java, a variable always points to the entire folder , and the bottom-up rule is always used to resolve method calls. Therefore, the dynamic type is InterestAccount after the assignment in your example has taken place. (In languages with static dispatch , like C++, it is possible for a variable to point to an intermediate partition of a folder. This makes code written in the language much harder to understand, since the bottom-up rule only sometimes applies, and it leads to bugs, because it breaks encapsulation. Thankfully we never need to worry about such situations in CS 2110.)
