The heap property does not mean that searching the heap takes time proportional to its height. Remember that a heap is not a BST; a value less than a node could be in either the right or the left subtree, so you might have to search them both. This is more analogous to the trees in A5, where searching algorithms are linear in the number of nodes.

You want a "dictionary" or "map".

An entry (v, i) tells you that value v is in array element b[I], where b is the array containing the heap.

Therefore, you can find out where a value is in array b in expected constant time --you'll learn about hash tables and hasp maps later.

By "maintaining the map" we mean the following. Suppose value 20 is in array element 0, so the dictionary has contains the pair (20, 0). 20 is in the root of the heap. If 20 is bubbled down to array element 16, then the dictionary has to be changed to contain the value (20, 16).
