In a generic context, a "hash table" can refer to an implementation of either the Set or the Map abstract data type based on hashing keys (the topic of next Tuesday's lecture). Whether or not values are paired with keys (Map vs. Set) is of minor importance compared to the general strategy of using hash codes.

In the Java standard library, HashSet and HashMap are classes implementing the Set and Map interfaces, respectively. Hashtable (which did make an appearance in A6) is also an implementation of the Map interface, but it dates from Java 1.0 (before that interface was even defined; it was retrofitted to support it). Hashtable differs from HashMap in some of its concurrency properties (also a topic for next week); in that sense, it is analogous to Vector vs. ArrayList . We tend not to use these legacy Java 1.0 classes unless they're needed for compatibility reasons; HashSet , HashMap , and ArrayList all have clearer names.

A set is a bunch of distinct values.

A map is a bunch of pairs, say (key, value)

A Hashtable is a special implementation of a set.

A HashMap is a special implementation of a map.
