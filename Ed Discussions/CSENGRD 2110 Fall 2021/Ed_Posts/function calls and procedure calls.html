If you search on the homepage of JavaHyperText, you can read its definition of "procedure" and "procedure call," which may help clarify things.

In this course, a "procedure" is a method that does not return a value; therefore, procedures can be identified as methods whose return type is void . If a method is to be useful, yet does not return a value, then it must have some kind of side effect. Often, that side effect is mutating the fields of the object that the procedure was invoked on.

c1.increment() is a procedure call because method increment() in class Counter has a void return type. Its side effect is incrementing the field count in the object referenced by variable c1 .
