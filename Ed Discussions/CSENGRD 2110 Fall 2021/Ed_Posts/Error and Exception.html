Here are the facts. Read carefully. 1. Any object of class Throwable or its subclasses can the thrown in a throw statement. 2. Any object of class Throwable or its subclasses can be caught using a try-statement. Below, T can be Throwable or any of its subclasses: try { code a try block } catch (T e) {code in a catch block} 3. The TWO direct subclasses of Throwable are Exception and Error. By points 1 and 2, they and their subclasses can be thrown and caught. 4. However, it makes no sense to try to catch Error and its subclasses because their nature is that the program cannot be continued properly. For example, one error is OutOfMemoryError (or something like that). 5. Catch an Exception object (or one of its subclasses) if at the level that it is caught, something reasonable can be done about it. That's enough for now.
