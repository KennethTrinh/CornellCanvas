This can happen if you try to use the old JUnit 4 Test annotation instead of the current JUnit 5 ("Jupiter") Test annotation. Did you follow the directions exactly when creating your ElephantTest class? Try following them again and compare the resulting import statements to what you currently have. At a lower level, the error is telling you that, when we try to run your ElephantTest , your code tries to use Java's "reflection" features to query all of an object's members (both public and private). But we run your code in a "sandbox" (implemented using Java's SecurityManager), and such behavior is disallowed. Since you presumably aren't trying to use reflection yourself, this must be coming from the JUnit infrastructure. We allow this for JUnit5 classes, so if access is being denied, you must be using something other than JUnit5.
