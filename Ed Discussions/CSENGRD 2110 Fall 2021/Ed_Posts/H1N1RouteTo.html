In the method's specification (#2), it says : 

2. The ONLY case in which this method must create a list object (using
        // a new expression new ArrayList<>() or new LinkedList<>()) is the base
        // case, when c is the root. Do NOT create a new list object using a new-expression
        // in the recursive case.

So, in the base case, it should actually create a new list object and not just declare its type. I think it would be important to consider what is done with this new list in the base case and also to remember that this method is returning a List<Person> object.
