In preparing for the final, I came across this statement: To make several instances of a class execute concurrently, it is sufficient to implement interface Runnable.

My understanding after reading the Javadocs for Thread is that the class must also implement method run(). Is this understanding correct?

One thing that might be confusing you is that, if you write a class containing instructions that you want to execute concurrently, and you decide to extend class Thread , then you must override Thread 's run() method and put your instructions there.

However, the more flexible way to do this is to write a class that implements Runnable , in which case the compiler will require you to override run() . Then you can pass instances of that class to the constructor of generic Thread objects.

Interface Runnable has one abstract method, run(). Therefore, if a class implements Runnable, it must override method run().

Thus, you are right; run() must be declared in the class.

However, we don't have to mention it because if a class implements an interface, it must override all methods declared in that interface.
