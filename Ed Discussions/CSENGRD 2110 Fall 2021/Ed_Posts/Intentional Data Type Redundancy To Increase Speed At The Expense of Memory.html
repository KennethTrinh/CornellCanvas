I was looking at the lecture 8 slides and found it interesting how Array Lists and Linked Lists do the same thing, but their methods have different efficiencies.

Is it at all common to, sort of, combine data types into one data type to incorporate the best aspects of each type?

Like if I wanted a data type that takes constant time for both prepending and getting would it make sense to create a new datatype that holds an Array List and Linked List version of our data, and then have getting and prepending methods that only call the faster prepending and getting methods of each sub datatype? And then maybe a third method that makes any changes to one sub datatype carry over to the other sub datatype when switching from prepending to getting?

Is this a thing?
