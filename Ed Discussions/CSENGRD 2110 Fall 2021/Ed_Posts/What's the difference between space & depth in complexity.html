Author: David Gries (admin)

To sort an array of size n, the first quicksort algorithm we wrote has recursion depth of O(n), which means that O(n) frames for calls will be on the call stack at some point in the worst cast. This is extra space used by the quicksort algorithm. Therefore, the worst-space requirement of quicksort is O(n). However, by rewriting the algorithm to use some iteration instead of recursion, this can be reduced to worst-case depth O(log n), so the better quicksort algorithm requires O(log n) space.

Author: Curran Muhlberger (admin)

Vote Count: 0

Space complexity refers to how much memory an algorithm requires in order to do its worn, beyond that taken up by the inputs themselves. For recursive functions, some space will always be required for the call stack. Therefore, the space complexity for such functions is at least as much as the depth of recursion. However, some functions also use auxiliary space in other ways. Merge sort, for example, allocates a temporary array to store half of the values it is trying to merge - it is not an "in-place" algorithm. The size of this temporary array contributes to the space complexity of merge sort. Typically, the total space complexity will be the larger of the max recursion depth and the max size of any auxiliary arrays allocated. (as an aside, the "merge" operation does not need auxiliary space if its inputs are linked lists rather than arrays; try it out!)
