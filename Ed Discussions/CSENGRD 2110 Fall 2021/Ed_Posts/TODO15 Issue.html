Author: David Gries (admin)

This is typical of how navigation on a save works. It allows you to click on folders to get to the folder where you want the file saved.

You CAN click on a greyed out file, but be careful, when you do that, the "Save As" file name is changed to the name you clicked on, and saving will store in that file. If it's where you want to append, fine. If not, you'll mess up that file.

Instead of clicking on a file name, just put in the Save As field what the name under which you want to store the file.

Author: Curran Muhlberger (admin)

Vote Count: 0

This is a quirk of the default Swing look-and-feel on Macs. On other platforms, the default look-and-feel is a pure-Java theme named "Ocean/Metal", which is designed to behave the same on all computers (and its "save" dialog behaves as expected). It's possible to use this LaF on Macs as well, but by default it tries to use "native-looking" widgets instead. Unfortunately, a quirk of that is that the default "save" dialog seems to gray out existing files.

There is probably a way to get the Mac save dialog to behave more normally, but I'm afraid I don't have a Mac myself to explore. It's not needed for this assignment (the default dialog is all we're looking for), but if you're curious and figure out how, please share it here!
