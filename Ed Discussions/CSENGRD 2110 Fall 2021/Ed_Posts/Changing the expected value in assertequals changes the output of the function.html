Get names returns with spaces included in between the elements. for an array with elements A then B then C for example it returns a string [A, B, C]. Kind of annoying but you should have looked at the spec and/or the code because its kind of obvious. Also thinking of tostring representations they usually include the spaces actually, I think most array toString use the spaces. Lowkey I did make the same mistake as you at first so I get the issue. You can see from the expected vs. actual there are extra spaces in the actual and none in the expected when you run the assert equals.
