ANSWERS

-3 Typo on page 2 about assert statements . You see this:

assert n != null && name.length() >= 1;

It should be the following, where n is the parameter for the name of the elephant:

assert n != null && n.length() >= 1;

-1. When I create a JUnit testing class and then put in an assertEquals or assertThrows statement, I get an error.

Press your mouse over the red x before a line with an assertEquals or assertThrows call that shows an error. A window will pop up suggesting the need for an import statement. Select that and let Eclipse put in the import statement.

0. What if my files are not in package a1?

You must put the .java files in package a1; otherwise, your classes will not compile with our grading program and we have great difficulty grading it.

To create the package, select directory src in the ExplorerPane and do menu-item File -> New -> package. Once it is created, drag your two .java files to into that package. You may then have to put a package statement at the head of each .java file.

1. How many times can I submit on the CMS ? Submit as often as you like. Only the last one is graded.

BUT DO NOT take this as an opportunity to submit every 3 minutes, submitting 15-20 times. Submit when you know it is done, but if later you see how to do something better, submit again. Don't choke up the system by wantonly submitting every two seconds. Note: the auto-helper will grade only 3 submissions for your group each day.

2. I get a warning about my method specification because I am not using @param or @return . What do I do?

Ignore the warnings and delete those @param thingies for now. We don't need them.

Read about method specifications in our Style Guidelines: Visit JavaHyperText page and click "Style Guide" in the left part of the top navigation bar. 3. Should we check for situations like if an Elephant is its own child or if an Elephant is awarded in a year that is < 0 ?

No. Please implement the spec of each method exactly; do nothing more, nothing less. We are not attempting to create a completely realistic class to be used in a real live situation. We are attempting to teach you about OO in Java with a minimum amount of your time.

4. Can we add other methods to class Elephant ? Do not add other methods. There is no need for them.

5. If I want to call constructor 1 from a constructor 2, I can't put assert statements in the beginning of constructor 2. What do I do? If you are not contemplating calling one constructor from another, which hasn't been taught as of this writing, no need to read further.

But the answer to this question appears in note #106.

6. Suppose my field for the number of children is nchilds. Since nchilds is private, from Elephant object e (say), how can I reference/change nchilds, the number of children in Elephant object d?

Just do it; you will see that it works. Just use nchilds to reference e's field nchilds. Any method in class Elephant can reference private fields and methods of any object of class Elephant.

7. May I use fields in ElephantTest to reduce the number of declarations needed in methods in this class?

No, don’t do that, for at least three reasons.

  • You don’t know enough about JUnit testing to do this right. For example, you don’t know the order in which methods are called.

  • Such fields often create interactions between testing procedures, but those testing procedures should be completely independent of each other.

  • A declaration may be far from its use, making it more difficult to understand the methods.

    • Remember, once you write a declaration-assignment, you can easily copy-and-paste it into another method.

    8. How can I write this in Java? "this Elephant and Elephant e are different"?

    Keyword this is a pointer to the object in which it appears. You will learn this soon. So you can write this expression as

    this != e
