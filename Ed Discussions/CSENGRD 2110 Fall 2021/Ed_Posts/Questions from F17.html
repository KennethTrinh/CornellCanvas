For your second question, it looks like the first for loop counter is being doubled after every iteration, so that will give you log(n) time. In the second embedded for loop, it looks like a regular loop that iterates over n elements, so it would be n time. Since it's a nested for loop you multiply the two times together and you get O(nlogn)

Just did this question last night. isSameTree method will return false if both both of the trees don’t share the same structure. Let’s say that we have tree A and tree B.

Tree A looks like this:

A B C

Tree C looks like this:

A B

Once the method detects that one node is null in the second if statement, it will return false. That means that the method looked at the minimum nodes of tree C.
