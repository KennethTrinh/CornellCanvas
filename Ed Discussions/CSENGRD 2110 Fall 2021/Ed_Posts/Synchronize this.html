Every Object in Java has an associated "monitor", which is the infrastructure needed for locking and waiting. Synchronizing on an object synchronizes on that object's monitor, and nothing more. It implies nothing about any of the fields in that object. It enables coordination only between threads who synchronize on the exact same object at runtime.
