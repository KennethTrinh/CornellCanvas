Bugs in GUI programs can be difficult to catch because sometimes they appear to work under some operating systems but not on others. However, if even just one OS shows unexpected behavior, then there is almost certainly a bug in the program (and not in Java or the OS). Note that, when paintComponent() is called, the Graphics object is not guaranteed to be "erased", and an "opaque" component like JPanel (which GameComponent inherits from) is required to draw something on every single pixel inside of itself. If you don't, then the un-erased content that shows through could be random and certainly OS-dependent. If you implement TODO 4 correctly, then you should automatically satisfy the "opacity" requirement, but if you forget a step, then behavior like what you describe is expected.
