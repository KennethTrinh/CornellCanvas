That is a warning that may show up in Eclipse, but it is not an error. Java made a decision early on that it should be very easy to "serialize" objects - that is, save them to a file or send them over a network, so that an entire application's state could be recovered on another computer or at a future time. They were so keen on the idea that all Swing components were made serializable, so you could checkpoint and restore an entire GUI application without inventing your own file format. However, serialization turned out to have complications, especially if classes could be extended in the future. They realized that class authors needed to provide additional information - like a serialVersionUID - in order for it to work properly. Since GameComponent extends JFrame , it is bound by this contract. However, we're not discussing serialization in this class, nor do we expect anyone to try to serialize their Click-a-Dot game. So you can ignore this warning.
