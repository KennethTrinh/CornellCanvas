Consider a specification of a public method. A "client" will see it ---someone who wants to USE your class but doesn't get to see the code. Therefore, that specification should not mention anything by name that is private, because they don't know what it means. That is all.

Consider class String. You can see its specification here:

https://docs.oracle.com/javase/8/docs/api/java/lang/String.html

You don't see any code, just the specifications of methods. Here is one constructor:

/** Coonstructor: a new String it represents the sequence of characters in array value */ String (char[] value)

Now, the new String of chars has to be stored SOMEWHERE within the object, but the spec doesn't say where. For example, it doesn't say

"Store the string of values in field c."

That would be giving away unnecessary information.
