NO. Do not use try-catch. There is absolutely no reasonable use for it in A1.

In areSibs, the spec is:

/** Return value of "this elephant and e are siblings." (note: if e is null they can't be * siblings, so false is returned). */

Look, if e is null, false should be returned. Implement that.

There is nothing to assert, since there is no precondition.
