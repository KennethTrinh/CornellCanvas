Sloppy proofreading - it should say if (person.equals(p)) . In this context the == happens to work because only one Person object is ever created with the same name in the H1N1 app and in the test cases (note the use of static fields like personC in the test, rather than new expressions). But because Person 's superclass overrides equals , the use of == should be considered a bug. Sorry about that - feel free to fix it in your submission.
