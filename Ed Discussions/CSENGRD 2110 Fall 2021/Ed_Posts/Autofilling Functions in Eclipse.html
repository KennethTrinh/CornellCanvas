For Eclipse to provide such documentation, it needs access either to the source code of the classes or to the generated JavaDoc. These are sometimes, but not always, provided alongside your JDK installation. Do you see a pattern for when you do or don't see documentation? Do you always see it for classes that are part of a demo or assignment, but never for classes in the Java standard library? If so, then you probably need to install the Java sources or documentation. I admit I'm not an Eclipse expert (and it's been more than a decade since I was actively trying out different JDKs on my machines), but this might be a direction to search in. I suspect the issue has more to do with the availability of parts of the JDK and less to do with Eclipse's preferences.
