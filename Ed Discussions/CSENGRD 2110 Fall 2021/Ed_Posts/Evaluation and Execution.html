Please read step 1. It gives an example:

C@48ae

That's what you should put on the tab of the folder. The "C" is the class name, the @ sign stands for "at", and 48ae is either the address in memory of the object (if the computer is evaluating the new-expression) or a number that YOU chose (if you are evaluating the new-expression.

You don't "assign" anything to the new object.

There are several ways we might use this expression.

1. In an assignment like. v= new C(...). Evaluation of the new-expression could yield the value C@48ae; that value is then stored in v.

2. In an expression, like. (new Fraction(1/3).toDouble() In this case, the new-expression is evaluated to yield something like Fraction@5. Then, immediately, that object's function toDouble() is called to give that fraction as a value of type double. The object can't be referenced after this!

I hope this helps.
