Binary search through an array is O(log n) because finding an element given an index i is O(1). Remember that indexing an array is O(1). Indexing a doubly linkedlist is not constant time. Going to a particular index i will take i/n-i traversals(whichever one is less), since there's no way to simply jump to a specific index. We would have to start at the tail or head and count our way to a specific index(doing something like node = node.next or node = node.prev inside a loop most likely if that helps). So definitely O(n).
