Java supports a limited form of type inference when constructing generic classes. If the type parameter's value is "obvious" from context (for example, assigning to a variable with static type ArrayList<Review> ), then you can use the "diamond" in place of the type. Search for "diamond" or "<>" in JavaHyperText, or read this description of the feature from when it was first introduced: https://docs.oracle.com/javase/7/docs/technotes/guides/language/type-inference-generic-instance-creation.html .
