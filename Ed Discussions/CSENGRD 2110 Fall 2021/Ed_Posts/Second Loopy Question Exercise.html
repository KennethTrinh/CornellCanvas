The loop condition B is such that when !B && P => R is true, then execution of the loop stops. In this case, if k were to equal m (as the loop condition) , then intuitively, R is satisfied, so we should exit the loop. We wouldn't even need to increment k at that point or make any progress towards satisfying R. If k != m (or specifically, k>m), we can make progress to satisfying the loop condition.
