I'm having an issue with the tostringbrief representation of my tree for the test cases of my insert method. For example, when I assertEquals for the creation of Tree 1, I compare it to "A[B[D E F[G[H[I] J]]] C]" and this passes. When I assertEquals for Tree 2, since the only difference is the replacement of personH with personK, I would expect the code "A[B[D E F[G[K[I] J]]] C]". However, if i compare to this representation, an error is returned and the program states that it returned "A[B[D E F[G[J K[I]]]] C]". These two trees are the same when done out, however because the string representations are different the asserts keep failing. I'm not entirely sure why this is happening and was wondering if I could receive some guidance.

Author: Rebecca Liu (admin)

Vote Count: 1

toStringBrief includes the call selectionSort1 (childs) to sort the children by name before iterating through them (in this case, it would be in alphabetical order). That is why H[I] appears before J in the case of using makeTree1(), while K[I] appears after J in the case of makeTree2().
