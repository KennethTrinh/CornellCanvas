This is an example that they brought up in the concurrency lecture. Basically, x=x+1 is technically broken up into three instructions: 1) grabbing the data of x from RAM; 2) incrementing the value of x; 3) storing the value of x into RAM. If two threads are running the same instruction concurrently, the case may appear where one thread interrupts another thread during the instructions, which would cause a race condition. So x=x+1 is the thread operation in this case.
