If the default value for a field is what is needed to truthify the class invariant, then just use the default and there is no need to store a value in the field in the constructor.

For example, my solution has the following. Since the default for type int is 0 and initially the child has no children, there is no need to have an assignment to children in the constructor. But of course, in your testing of the constructor, you must still test that children contains 0.

/** Number of known children of this elephant. */ private int children;
