For a perfect tree, it’s 2^(h+1)-1

The worst case for a binary tree is O(n) because it’s basically analogous to a linear search (the node could be anywhere in the tree). For a binary search tree, it is O(log n).

Author: Varun Gande (student)

Vote Count: 0

O(log n), where n is the number of nodes, is the best case if your binary search tree is balanced. The worst case is when the binary search tree degenerates to a list, and it’s effectively like searching through a list (hence O(n)).
