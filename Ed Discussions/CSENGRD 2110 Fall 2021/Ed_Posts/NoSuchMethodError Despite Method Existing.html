Also, keep in mind that passing all of your own tests does not imply that your code has no problems, especially if it needs to be integrated into a larger software system. Unit tests are only one stage of software verification; integration tests are the next level, and that is what the AutoHelper provides. It is very easy to violate an interface specification in both your implementation and your tests, which is why integrating with independently-authored code that uses the same interface is so useful for finding bugs.
