Hi, Class E cannot access Class F, but Class F can access Class E. So if E implements K2, it must include every method of K2 as it cannot access p() in Class F. Because Class F can Access its super class, Class E, it can use E's n() method in order to fully implement K1, K2 as it as not itself implement n(). I recommend playing around in Eclipse and copying the code in order to get a better understanding of how this works. interface K2 { public int n(); public void p(); } interface K1 { int n(); } class E implements K2 { public int n(){} public void p(){} } class F extends E implements K1, K2 { } Notice how I implemented all of K2's method in Class E's interface and removed all the code of Class F's body and the code should work fine and not give out an error about implemented classes. (Ignore the error about how n should return an int, it just means n has yet to be coded with a return statement returning a type int.) If you were to call p() on a Class F object, it would look to the super class, in this case which is Class E, and use it's p(). However, Class E cannot access the methods of Class F.
