javax.swing.JFrame is a class in Java's standard library that represents a graphical window on your computer; we will discuss JFrame in more detail when we talk about Graphical User Interfaces (GUIs). In previous versions of this course, JFrame was used as a visual example in some early lectures, and as you saw, it is also used as an example in some JavaHyperText documents. The expression JFrame.class evaluates to an object representing the idea of "being a JFrame" (and not something else). I often think of it as being analogous to the string "JFrame" in that it says what an object's dynamic type is; however, unlike a simple string, a Class object is more robust and has additional properties related to Java's "Reflection" features (so beyond just storing the name of a class, it also lists what public methods are defined, etc.; it lets other code learn about the classes being used in a program and their features). For our purposes, we typically only use Class objects to ask whether two objects have the same dynamic type. For example, if you had an object ob , and its static type was just Object , but you wanted to know whether its dynamic type was really JFrame , you would write this expression: 
if (ob.getClass() == JFrame.class) { // ob is a JFrame }
If you're more comfortable thinking about names and strings, you could write something like this: 
if (ob.getClass().getName().equals("javax.swing.JFrame")) { // ob is a JFrame }
However, comparing Class objects directly is more efficient and more robust than comparing class name strings.
