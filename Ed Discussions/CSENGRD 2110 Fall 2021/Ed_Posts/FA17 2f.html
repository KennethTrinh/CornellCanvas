Yes, an abstract method declaration is sufficient for the compile-time reference rule to conclude that the call is legal. Because it is impossible to instantiate an abstract class or an interface on their own, the compiler is allowed to assume that whatever value is ultimately referenced by that variable at runtime will have a concrete implementation of the method.
