Just a general question about heaps: I remember in the CMS reading that array size were heaps were determined at runtime. However, at what point does the heap decide it needs to increase it's array size, and does it accomplish this by simply doubling the length and copying the contents over?

Author: David Gries (admin)

1. In class ArrayList, when the need is to add an element but there is no more room, yes a new array of at least twice the size is created, the old array contents is copied into the new array, and from then on the new array is used.

2. We can do the same thing in the implementation of a heap.
