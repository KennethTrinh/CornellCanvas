The second if-statement returns false if is it known that the left subtree is not a BST. That's the case if t.left is not null and the calls isBST(...) returns false. Therefore, when it comes time to test the right subtree. We know the tree is a BST if the right subtree is (or if it is null). Therefore return true (in the last if-statement) if the right subtree is null or it is a BST ...
