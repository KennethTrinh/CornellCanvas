You should test as many of your assert statements as you wish. Do enough so that you know you are on the right track.

But I don't know what Precondition you are testing in your example.

Let's suppose that, in addMom, you have the assert statement:

assert mom == null;

To test it, you have to create an Elephant object in which mom is NOT null:

1. Create an elephant e1. 2. Create an elephant m. 3. call e1.addMom(m) so that e1 has a mom. 4. Finally, have an assertThrows in which the anonymous function calls e1.addMom(m) .

The handout says, if you read it carefully, to do one of the following:

1. Write a fifth testing procedure to test all assert statements.

OR

2. For each method that has assert statements, add tests for them to the appropriate one of the four testing procedures you already have (for groups A, B, C, D).
