Suppose I have the following code:

LinkedList<Integer> intList = new LinkedList<Integer>(); LinkedList<Boolean> boolList = new LinkedList<Boolean>(); public static void foo(LinkedList<Integer> list){ for(int i = 0; i < n; i++){ boolList.add(0, intList.contains(5)); } }

I want to determine the tightest average complexity of the method foo().

I think it's O(n^2) because we add the O(n) complexities of add and contains inside this loop to get O(2n), multiply it by the loop time O(n) to get O(2n^2) which simplifies to O(n^2).

My friend thinks it's O(n^3) because we multiply: O(n) of the loop, O(n) of the add, and O(n) of the contains to get O(n^3).

In other words, I'm asking if two operations take O(n) time inside the same loop, do we add or multiply them?

The complexity is O(n^2) for a different reason from both given. Adding to the front of a LinkedList takes constant time since you only change a few pointers in the list in order to add the new Node. Think back to how you wrote the prepend method in your LinkedList assignment. Did that method depend on the size of the list? As a result, since contains takes O(n) time, add takes O(1) time and the loop iterates n times, the overall complexity is O(n^2)
