Start an Eclipse project, to be used play around when you want. Nothing big. Suppose you don't believe the reason why to make a class abstract. Well, put the following class into that project. No problem, right? well put "abstract" after public on the first line. What happens in Eclipse? Does a read x appear to the left of line 2, and what does it say? Then delete "abstract". This is perhaps simplest way you can make concrete for yourself that the purpose of making a class abstract is that you can't create instances of it --the new-expression for it is illegal. If you DON'T do this, you may forget the reason for making a class abstract. However, once you have done this, you probably will never forget it. This is what I mean by trying Java principles, rules, definitions out in Eclipse. It's not a big deal, but it can help a lot. 
      public class C {
          C var= new C();
      }
