Yep, you first identify h, which is hashCode%size, and then attempt to place the element in bucket h. It’s possible that bucket h is full and does not contain the element you want to add. So you continue checking buckets, until you either find an empty one or find the element itself. Linear probing tells you then to check h, (h+1)%size, (h+2)%size,…

Quadratic probing checks in this order: h, h+1^2, h+2^2, h+3^2(from JHT), also with wrap around like linear probing.
