Hi, I am confused why 2a is false.

According to Java Hyper Text,

"Throwing an Error causes immediate termination of a program. There is no chance for the program to handle it in some fashion."

The try-catch statement can be used to catch ANY thrown exception --any subclass of class Throwable. That includes Error and its subclasses. JavaHyperText does not mean to say it can't but only that in general, a thrown Error results in a real mistake that cannot be fixed.

Here's a case where we actually wrote a program to catch an Error, a StackOverFlowError. Do you remember the demo of method Power(b, n), to raise b to the power n? We had a slow method Power and a fast method Power. We actually had a method main that would call the slow method Power with increasing value of n until the call stack overflowed. This gave us a chance to demo vividly the O(n) power method and the O(log n) power method. So, even though in general an Error is not fixable, we used an Error to learn something.
