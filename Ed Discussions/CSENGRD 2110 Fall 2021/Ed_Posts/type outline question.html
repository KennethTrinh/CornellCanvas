Author: Curran Muhlberger (admin)

Any primitive type can be converted to another (possibly with a loss of information) by using a cast .

However, Java will also automatically convert some types to wider ones in certain circumstances. For example, when adding an int and a double , the int will first be converted to a double , and then the + operator will act on two double values. The rule is that all operands are converted to the type of the widest operand. This rule works well for int , long , float , and double .

For the other types, an additional rule comes into play. Those types don't actually have any mathematical operators of their own - there is no + operator that adds two byte s and returns a byte , for example. In those cases, Java again automatically converts operands to a wider type that does have the requested operation. The narrowest such type is int , so those types will all be converted to at least int in order to do math on them, even if there is no other int operand.
