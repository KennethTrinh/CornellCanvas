Yes, the Set interface considers two objects to be equal if their .equals() method returns true (and since .equals() must be symmetric, it doesn't matter wither it calls e.equals(o) or o.equals(e) ). The implementation of equals() is the one found via the bottom-up rule for the object it is invoked on (in this case, o , the argument passed to contains() ). So if you implemented o 's class and overrode equals() , then your method is the one it will call.

To clarify your follow-up question, this might not be the equals() method for type E , for two reasons: 1) e 's type may be a subtype of E , and 2) the parameter type for contains() is Object , so o might not be an E at all.
