I understand the process of string concatenation and why it takes time proportional to string length, but what if we decide to concatenate a several character string to a several character string instead, of a one-character string to a several character string?

Does the process of concatenation loop through each character in the appended string, doing single character concatenation on each successive character of the appended string until none are left? Thereby making (I think) time n^2.

Or does multiple character concatenation work in a different, more time-effective way?

When two strings are catenated, s1 + s2, a new String object is created with an array big enough to hold both strings. The string in s1 is copied, then the string in s2 is copied.
