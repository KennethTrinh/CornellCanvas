Title: Where to put assert statements when calling one constructor from another

Author: David Gries (admin)

Date: 2021-09-05T02:35:00.708483+10:00

Category: General

Vote Count: 0

This note has to do with calling one constructor from another ---for example, in A1, calling the first constructor from the second constructor. You don't have to read further if you are not calling one constructor from another. Rule : Within a constructor, a call on another constructor cannot appear in any place except as the first statement. This means that if you decide to have constructor 2 call constructor 1, the assert statements for Preconditions cannot be at the beginning of constructor 2; they must come after that call of constructor 1, e.g.:

public C { this (...); assert bool-exp; ... } That is OK. NOTE: If a call on constructor 1 does some assert-checking of Preconditions, the asserts in constructor 1 do not have to be repeated in the rest of constructor 2.
