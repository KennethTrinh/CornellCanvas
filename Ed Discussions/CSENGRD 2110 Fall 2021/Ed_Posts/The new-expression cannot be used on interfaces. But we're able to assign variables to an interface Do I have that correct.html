Suppose we have this: interface I1 {...} class C implements I1{} and somewhere we write I1 b= new C{...}; The object created looks as shown below. It has three partitions, for superset class Object, class C, and interface I1. In evaluating the new-expression new C{...} this object is created and a pointer to it is used as the value of the new-expression. Assignment to variable b requires an upward cast to I1, which is inserted automatically since it is an upward cast. Then the pointer to the object is stored in b. This means that b points at the whole object. However, since b's (static) type is I1, by the Compile-time reference rule, the only method calls like b.m(...) that can be compiled are ones in which m is declared in interface I1 or in class Object. 
       Object
          |   I1
          |  /
          | /
          C
