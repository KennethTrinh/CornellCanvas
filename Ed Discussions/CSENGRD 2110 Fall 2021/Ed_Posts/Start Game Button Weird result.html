Found the answer in my case, when calling fillRect() on the Graphics object, I had been using getX() and getY() for the positioning, with getWidth() and getHeight() for the size of the draw. Using the getX and getY methods put the origin for the draw in a strange place, and I didn't need to use them at all, since this is drawing relative to the Graphics component itself rather than to the whole JFrame. In short, the Graphics object should have it's own origin that doesn't need to be specially specified.
