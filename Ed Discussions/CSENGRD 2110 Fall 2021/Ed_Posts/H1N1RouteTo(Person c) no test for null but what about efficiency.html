I don't now what you mean by "The actual implementation really does not need a null test for correctness". There being no route to c certainly has to be tested. Also, A.H1N1RouteTo(null) should also be tested. Perhaps your question is about putting in a special base case if (c == null) return null; One COULD put this in to remove the task of traversing the whole tree if c is null. But I would NOT do this because the case would happen so rarely.
