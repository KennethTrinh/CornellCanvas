Based on the method specification, it seems state that two trees are equal as long as they have the same number and values of children. Is it correct to state that two trees are still equal even if they have different shapes, as long as they are of the same class, have the same size, and their children set are equal? In other words, order is not important when comparing H1N1Trees?

We define children as a SET of trees. Therefore, Trees T1 and T2 are equal if (a) They have the same person in their root node and (b) T1's set of children equals T2's set of children. We cannot talk about the "Shapes" of the children, because they are sets of children. And, therefore, order is not important because order is not important in sets. We could have defined the trees as Lists of children. Then, order would have been important.
