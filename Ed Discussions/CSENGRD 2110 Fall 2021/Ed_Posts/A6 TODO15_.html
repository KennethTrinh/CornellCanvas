Printing the exception type in the dialog text before the exception's message is an "embellishment" of the reference implementation. It is not required by the specifications, but you can add it if you think it makes the error easier to understand.

Most of the time in this course we emphasize strict adherence to precise specifications, but GUIs are a good place to express some creativity, so you're allowed some freedom in font sizes, font faces, color shades, and helpful bits of text here and there. Just make sure you don't contradict what is required by the specifications.
