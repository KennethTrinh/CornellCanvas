When reading on JavaHyperText about the shortest path algorithm I found something very confusing. The pdf says "Note that if w is in the settled set, like w1, nothing will change" after putting the frontier node into the settled set. However, in a situation like the following:

distance from v to a settled node s1 is 1

distance between v to a settled node s2 is 4

distance from s1 to frontier node f is 1

distance from f to s2 is 1

The shortest distance from v to s2(d[s2]) would change from 4 to 3 if f is added to settled set, so why does the pdf say nothing should be changed for settled nodes? It doesn't affect how the algorithm would be written, but I'm confused about the logic behind this line.

----------- REPLIES -----------

Author: David Gries (admin)

Vote Count: 0

For node s in the settled set, d[s] is the shortest distance from the start node to s over ALL paths from the start node to s.

Therefore, the situation that you describe cannot happen.
