Author: David Gries (admin)

Suppose there are n vertices. How many edges are there? At most 5n. Therefore, the adjacency list implementation requires only space O(n) (for the vertices) plus O(5n), which is a total of O(n). But the adjacency matrix representation requires space O(n*n). Further, many operations, like processing the nodes adjacent to a particular node, requires constant time --at most 5 nodes have to be processed. But to even find the nodes adjacent to a particular node with the adjacency matrix implementation requires time O(n) to find those nodes.

Author: David Gries (admin)

You misunderstand the word "dense". Type dense into JavaHyperText and you get what you see below. Please study this. A graph is dense if the number of edges is close to the maximum. A graph is sparse if the number of edges is close to the minimum. These terms are not well defined in the literature. So we use these definitions: A graph with n nodes is dense if the number of edges is proportional to n*n; it is sparse if the number of edges is O(n). A map of a city is generally a sparse graph because the number of edges leaving each intersection is generally 4 or less.
