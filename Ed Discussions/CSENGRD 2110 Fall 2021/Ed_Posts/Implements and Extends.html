Consider this situation:

1. C is an interface, E and F are class

2. F implements C

3. E extends F

Do we need to have all methods in C declared in E? Should I also write E implements C?

The best way to answer your question is to try it in Eclipse. Define interfaces and classes exactly as you describe here and see what happens. You will remember the rules much better that way.

For the formal answer, if class F implements C and class E extends F , then:

  • Class F must implement all methods declared in C (unless F is marked as abstract )

  • Because E extends F , E will inherit all methods declared in C , so it does not need to declare them itself (unless F is abstract , in which case E will need to implement any abstract methods left in F )

  • The compiler knows that E implements C because of subtyping rules: E IS-A F , and F implements C . Therefore, you should not write E implements C - that is redundant to the compiler.
