Why is it better to use:

Integer v= Integer.valueOf(9)

Instead of using the constructor?:

Integer v= Integer(9);

Author: David Gries (admin)

Yes, a new object is created. But, since Integer objects are immutable, there is no need

for more than one Integer object that houses the value 9.

Author: David Gries (admin)

Vote Count: 2

This is a rather new feature, that use of the constructor is discouraged.

One reason: This allows optimization. For a small int value like 9, they may create only one Integer object with that value and return a pointer to that one object. That can't be done if a constructor is being used.

Author: Dasha Griffiths (admin)

Vote Count: 1

Yes but that would be unoptimized since a constructor will do that for all cases. The valueOf method is optimized so that it will use the most efficient way to get the value you want to assign. While using valueOf vs the constructor might not matter for small cases like 9, it could make a large difference with values like 100000000000.

Author: Curran Muhlberger (admin)

Vote Count: 1

The optimization allowed by avoiding a new -expression is called "interning" (related to "memoization" - memorizing the return values of functions called multiple times with the same arguments). While both expressions will yield an object with the appropriate value, they correspond to slightly different commands:

  • A new -expression" says "I want you to allocate a new folder in memory and initialize it to this state"

  • A "factory method" like valueOf() says "I want a reference to an object representing this value; I don't care where it comes from"

    • The latter is less constraining and therefore opens the door to optimizations like interning.
