The specification for the fields is a little mathematical, but if you study it for a bit, you should see that h is the index of the "first" element in the queue (that is, the one that was added longest ago and would be the next one returned by a "get" operation). And the index of the last element in the queue (the one added most recently) is (h+n-1)%b.length . If you were to append an additional element to the queue, then the first element would not change, but the last element would. The indices will remain consistent as long as you update n , the total number of elements in the queue. If this queue had a "get" or "remove" operation (not shown), such an operation would return the element at index h and increment h by 1 to point to the next-oldest element. This is why h is not always 0.
