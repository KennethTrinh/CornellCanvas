QUESTIONS AND ISSUES

  • Why does it say my Elephant won't compile?

  • What does it mean when it says "Code-under-test wrote N bytes of extraneous output?"

  • What does it mean when it says "WARNING: The autograder encountered an error while running your submission. Your results may not be complete."

  • What does it mean when it says "Failure in testName(): null" (when checking your ElephantTest test suite)?

    • ANSWERS

  • Your class in Elephant.java must compile on our submission server. In most cases, if it compiles in Eclipse for you, it will compile for us, but there are a few mistakes that your Eclipse might not catch. In particular,

  • Your class must be in the package a1 (Note: because of how the AutoHelper is implemented, wrong packages may actually show up as errors in the AutoHelper itself; see question 3.)

  • Your class must be named Elephant (if you had to rename your file to upload it to CMS, then something is wrong)

  • Your class must use the exact names of the methods given in the A1 handout. Check carefully for typos. In addition, the number of parameters and their types must be as given in the A1 handout. (Note: because of how the AutoHelper is implemented, wrong method names will actually show up as test errors for Elephant, but for ElephantTest they will indeed lead to a compile error.)

  • Your source code must be encoded as UTF-8 (you should have already done this, but double-check "Setting up Java and Eclipse" on our Canvas home page). If you changed it to UTF-8, be sure to restart Eclipse. Then, check the specification of isOlder(). If it has a weird question-mark character symbol, remove it and search for other occurrences of this character in anything you copied and pasted directly from the pdf handout.

  • You must not attempt to import JUnitTest functions in your Elephant class; those should only be used in your ElephantTest.

  • Your code should not print any output to the console unless it is part of the assignment specifications. A program's console output (System.out, System.err) is part of its interface, and good interfaces conform to precise specifications. Make sure your code does not contain any println() calls unless we ask for them.

  • Check that your classes are both in the package a1. We will try to make this error signature clearer in the future.

  • Your test procedure named "testName" unintentionally threw an exception (violating an assertion results in a test case failure , with a helpful error message; throwing an exception results in a test case error , which currently is not accompanied by a useful message). You likely violated a precondition as part of your test case. Additionally, if your test suite was green when testing your own code, this further implies that your Elephant implementation is likely not enforcing its preconditions correctly (or that you don't have assertions enabled in Eclipse; see A0).
