Why should you need that? c and y are NOT program variables. They are "thought" variables used in describing the algorithm.

Look at the spec (revised as per the item in pinned note Assignment A2);\:

Any positive integer n can be written in this form: c*2^y, where c > 0 and c is odd and y >= 0. Return the value c.

Your program does not have to have a variable x. Suppose n is 5 * 2^4 = 5*16 = 80. Then return the value 5.

Now look at what the hint in the method says to do:

Write a while-loop that continually divides n by 2 until n is odd. Then return n.

Suppose n = 2*2^4 = 80. If you divide n by 2 4 times you get n = 5*2^0 = 5, which is the value that should be returned.
