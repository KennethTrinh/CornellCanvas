For reference, UTF-8 is an encoding of Unicode characters. Unicode contains characters and symbols from pretty much every language - tens of thousands of them (this is in contrast to ASCII, which only contains about 100 characters common to Latin-based languages).

An encoding is how characters get represented as bits when they are written to disk. UTF-8 is the most common and convenient Unicode encoding, with many Latin characters requiring only 8 bits to encode them, and it is the default file encoding that Java assumes on Mac and Linux. However, many other encodings have been used throughout history, most of which can't represent all of Unicode, and they use different bits to represent less-common characters (so if you assume one encoding but are given another, you'll misinterpret the file's contents). Java assumes one of these legacy encodings by default on Windows, which is why we ask everyone to confirm that their Eclipse is set to use UTF-8 instead.
