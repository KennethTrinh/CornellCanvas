The command demonstrated in lecture was /open ; it takes one argument, a filename (like "Fraction1.java"), and it acts as if you had copy-pasted the entire contents of the file into the terminal. JShell can also make use of pre-compiled Java classes and libraries (".class" files) as long as they are on the "classpath." However, it does not support classes in the "default package," so you must put code in a package for this to work (this is why all assignment code is in a package named after the assignment). This is a more advanced, but more flexible, way of working.
