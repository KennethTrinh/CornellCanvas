F(0) is called because when F(2) is called, it will call F(2-1)=F(1) and then F(2-2)=F(0)

If it helps you could visualize the calls in the structure below (this doesn't show the order, just the calls):

F(3) / \ F(2) F(1) /. \ F(1). F(0)

F(1) would not return zero, it should return 1 in the base case:

if (n==0 || n==1) return 1;
