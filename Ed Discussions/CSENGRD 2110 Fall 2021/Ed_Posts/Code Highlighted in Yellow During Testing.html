That color highlighting represents a "code coverage" analysis. There are various degrees of code coverage, including:

  • Line coverage (every line is green or yellow; no red lines)

  • Branch coverage (every if-statement or conditional expression is executed with both a true and a false argument across the test suite; may consider short-circuit clauses as well)

  • Condition coverage (every clause in a Boolean expression is evaluated to both a true and a false across the test suite)

  • Path coverage (every combination of all the conditional expressions in the program is tested) - impractical in practice

    • Eclipse is most likely indicating branch coverage, which is a reasonable metric to target (the cost/benefit ratio is rather poor past that, though condition coverage is required in some safety-critical domains). If you can make your yellow lines green by adding additional tests that make any if-statements true in some tests and false in others, that is a good thing to aim for. But if you get stuck or it starts to get overly tedious (maybe Eclipse is actually measuring condition coverage), then just make sure you at least don't have any red lines.
