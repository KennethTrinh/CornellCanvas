Yes - JShell can make use of any Java classes or libraries that have been compiled and are in its "classpath" (they also can't be in the default package, which is why we require that all assignment code go in a named package). For example, if your project is stored on your disk at /home/netid/workspace/A3 (and assuming you saved your code in Eclipse and it compiled with no errors), then you can start JShell with the following option to play around with your project code: jshell --class-path /home/netid/workspace/A3/bin . Once you're in JShell, you may want to import a3.A3 for convenience.

An alternative approach is to ask JShell to interpret the contents of your source files without first compiling them, as we do for the lecture demos. This is done with the /open command within JShell; e.g. /open /home/netid/workspace/A3/src/A3.java .
