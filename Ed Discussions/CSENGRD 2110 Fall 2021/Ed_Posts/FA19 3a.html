1. You can take a look at the API for getClass(): 
https://docs.oracle.com/en/java/javase/11/docs/api/java.base/java/lang/Object.html#getClass()
Basically is that getClass() will return you the class itself. Similarly, you will get true for "a".getClass() == "b".getClass() because they are both instances of String Class(String is not a primitive type) 2. Yes. You want to use the equals method in class Color to make sure they are the same by the definitions in that Color class. And String is not a primitive type in Java, it is a class, and there's no wrapper class for that.
