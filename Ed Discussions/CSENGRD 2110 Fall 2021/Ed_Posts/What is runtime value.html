    Yes, the type of an expression in Java is independent of the runtime values of any of the variables involved. The types of all expressions are known at compile-time and checked for consistency then.

    (You can try to fool the type-checker by using casts, in which case Java adds runtime checks to preserve safety - if the runtime type of the value doesn't match the cast, then Java will throw an exception rather than executing any expressions affected by the assumed type.)
