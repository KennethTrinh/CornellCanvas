1. This is OK. 2. This is wrong. You don't know it is going to fill in values. If the call is new C(...), you should write: Execute the constructor call C(...) . 3. This is wrong. There is no variable to store it into. all you have is a new expression. Suppose the new-expression was used like this: (new C()).toString(); The value of the new-expression is not being stored anywhere. Here is how to write step 3. Use as value of the new-expression a pointer to (or the name of) the new object.
