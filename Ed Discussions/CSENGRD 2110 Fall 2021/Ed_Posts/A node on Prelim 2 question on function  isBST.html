Prelim 2 had a question with the spec and method header given below (note: this was on one of the prelims; the other prelim had a similar question, but slightly different; what we say here applies to both): /** Return true if the tree rooted at this node is a binary search tree (BST) and all its values are in the range h..k. */ public boolean isBST(int h, int k) PART of several incorrect solutions were similar to the following: if ( left.val >= val || val >= rite.val ) return false; return left.isBST(h, k) && rite.isBST(h, k) This attempt to test whether the values satisfy the BST properties is fundamentally flawed. It does not work. To see this, look at the 4-node tree shown below. It is not a BST because not every value in the left subtree is smaller than 5. Yet, the incorrect solution, with h = 0 and k = 10, would say it is a BST. Instead, this PART of the solution has to be written to test that every value in the left subtree is < val and every value in the right subtree is > val. Since the values are ints, we write this PART as follows. return left.isBST(h, val-1) && rite.isBST(val+1, k) This mistake turned to be a 4-point deduction, meted out consistently to all who made this mistake.
