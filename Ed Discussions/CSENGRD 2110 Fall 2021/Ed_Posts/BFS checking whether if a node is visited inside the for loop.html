If I have this code:

void bfs(Node start){ instantiate Queue named queue; queue.add(node); while(queue is not empty){ Node n = queue.remove(); n.visited = true; for(Node nb: n.neighbors){ if(!nb.visited) queue.add(nb); } } return false; }

Where I check whether or not a node was visited inside the for loop instead of before the for loop, would it still be correct?

This loop is not correct.

After removing n from the queue, one must check whether n.visited is true, and if no, do nothing; only if it is not visited should one proceed to set its visited status and put its neighbors on the queue.
