There's another example in Fall 18 Final where we do increment in the hasNext() method. I think that's because the HashSet is being implemented as an array. There's some null or unoccupied spots in there so we skip them by incrementing the index in hasNext(). The official incrementing happens in next() but hasNext() increments to deal with "1-off" situation or when there's an null at an index in the array. In this example, I see index only changing in next() so inside the hasNext() we're only checking if we've exceeding the ArrayList stockItems range.
