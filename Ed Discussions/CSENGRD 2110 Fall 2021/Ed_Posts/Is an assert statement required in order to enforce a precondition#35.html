Author: Curran Muhlberger (admin)

Vote Count: 0

Assert statements are executable code - they can go in constructor and method bodies, but not with field declarations. Unlike in some languages, they do not enforce constraints on a field whenever that field is changed; they only perform their check when they are executed. Additionally, assert statements should not be relied upon for validating user input. Remember - they are completely disabled by default, so a function's behavior must not rely on them (we violate this principle in assignment A1 for pedagogical reasons, but after that, assert statements take on a much more narrow role). Assert statements are most helpful for finding logic errors in the implementation of a class or library - an implementer uses them to confirm that things they think should always be true really are. Since a precondition is something that an implementer assumes to be true, assert statements are a great fit for verifying preconditions during development.
