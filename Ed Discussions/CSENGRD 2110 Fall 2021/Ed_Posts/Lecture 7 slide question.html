The compile time reference rule tells you what methods are able to be called (NOT which one is called). Since no method accrueInterest exists in class Account (the static type of b) or any superclass, b.accrueInterest() cannot be legal.

However, for the method printStatement(), there exists a method called printStatement in class Account. So a method call b.printStatement() is legal. Which rule tells you which version of printStatement is used?
