Author: Curran Muhlberger (admin)

There's a bit of nuance here. There is likely no performance difference between direct field access and a trivial getter/setter - the compiler will almost certainly inline such method calls. Using observers internally may make the code easier to refactor, and using private mutators could avoid bugs by ensuring that invariants are always maintained. On the other hand, there could be unexpected performance pitfalls if the observer is non-trivial (possibly as a result of refactoring). In general, when implementing a method, you are allowed to reference fields directly, and some methods (like observers and mutators) certainly must do so. But as code gets larger and higher-level, there is an "art" to deciding when to invoke other methods vs. going straight to the fields. You will get a feel for these tradeoffs through experience; for now, stick to the advice in the assignment prompts.
